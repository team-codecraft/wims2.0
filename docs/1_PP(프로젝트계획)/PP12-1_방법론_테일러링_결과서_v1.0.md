1\. 개요

1.1 목적

본 문서는 WIMS(Window & Curtain Wall Information Management System)
시스템 개선 프로젝트에 적용하는 SW 개발방법론의 테일러링(Tailoring)
결과를 정의합니다.

특허청 SW 개발방법론(객체지향 CBD 기반, ISO/IEC 12207 준용)의 7단계
프로세스 및 표준 산출물을 기준으로, PP11-1 TFT 구성계획서 v2.0에 정의된
프로젝트 거버넌스(인력 구성, 스프린트 체계, 리스크·보안·변경관리 등 16개
섹션)를 반영하여 산출물 및 절차를 조정한 결과를 기록합니다.

1.2 적용 범위

본 테일러링 결과는 WIMS 시스템 전면 개선 개발의 전체 범위에 적용됩니다.

> ① 제품 관리 시스템 (BOM / 자재 / 공정 / MES 연동)
>
> ② 견적 설계 시스템 (CAD 웹뷰어 / 도면 매핑 / 견적 산출)
>
> ③ 발주 관리 시스템 (계약 확정 / 최적 절단 / 발주서)
>
> ④ 제조 관리 시스템 (제작도 산출 / 절단 지시서 / 추가 발주)

  **② 중소 규모 사업    8개월, 10명 TFT → PP11-1 v2.0에 정의된
  (10명 TFT)**          역할(PM/BA/UI·UX/FE/BE/퍼블리셔) 기반 핵심 품질
                        산출물 중심 운영

  **③ 2-Phase 순차 오픈 Phase 1(S0\~S3, BOM+MES) / Phase 2(S4\~S15, 전체)

  **④ MES DB 직접       REST API 없이 DB 직접 연동 → 인터페이스 설계를
  연동**                API 명세 대신 DB 뷰/테이블 구조 설계로 대체

  **⑤ 애자일 요소       CO 단계에서 2주 단위 스프린트, 스프린트 종료 시
  병행**                현업 참여 데모 실시 (PP11-1 §5 스프린트 운영 절차
                        준용)

  **⑥ PP11-1 거버넌스   PP11-1 v2.0의 리스크관리(§8), 보안관리(§10),
  통합**                변경관리(§11), 에스컬레이션(§7) 절차를 테일러링에
                        반영하여 별도 관리 계획서 생략

5\. 단계별 Gate 기준

각 단계 완료 시 게이트(Gate) 검토를 실시합니다. Gate 3(Phase 1 오픈)과
Gate 5(Phase 2 오픈)은 핵심 게이트(★)로서 사업주(유미숙) 참여 검토를
의무화합니다. PP11-1 §7 에스컬레이션 절차와 연동하여 Gate 미통과 시
대응합니다.

  ---------- ------------- -------------------------------------------------
  **Gate**   **시점**      **진입 기준**

  **Gate 1** **AN 완료**\  • 요구사항 정의서 사업주 승인\
             S1 (M1 말)    • MES 연동 테이블/뷰 구조 MES 팀 서명\
                           • As-Is/To-Be 업무흐름도 확정\
                           • 총괄 테스트 계획서 작성 완료

  **Gate 2** **DE 완료**\  • BOM 논리/물리 ERD 확정\
             S2 (M2 1주차) • MES 연동 뷰 DDL 확정 및 MES 팀 서명\
                           • 화면 설계서 확정\
                           • 데이터 전환 매핑 정의 완료

  **★ Gate   **Phase 1     • Phase 1 통합 테스트 100% 통과\
  3**        오픈**\       • MES 연동 운영 검증 완료 (MES 팀 서명)\

                           • Phase 1 운영 이관 계획서 확정

6.1 PP11-1 v2.0 섹션별 테일러링 반영

  ------------ ------------------ -----------------------------------------
  **PP11-1     **관리 영역**      **테일러링 반영 내용**
  섹션**                          

  **§5**       **스프린트 운영**  16 스프린트(S0\~S15) 2주 단위, 스프린트
                                  종료 시 현업 데모 의무화 → CO 단계 산출물
                                  스프린트별 누적

                                  산출물 품질 책임 명확화

                                  Gate 미통과 시 에스컬레이션 절차 연동

                                  리스크 관리 계획서 생략 근거

                                  정책 정의 → 별도 보안 관리 계획서 생략
                                  근거

  **§11**      **변경 관리**      변경요청서(CR) 기반 절차, 영향도 분석
                                  의무화 → 테일러링 변경 시에도 동일 절차
                                  적용

  **§16**      **산출물 추적표**  23개 산출물 × 방법론코드 매핑 → 본 문서
                                  산출물 목록(4장)과 정합성 유지

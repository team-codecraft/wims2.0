# **1. 프로젝트 개요**

## **1.1 배경 및 목적**

WIMS(Window & Curtain Wall Information Management System)는 창호 및 커튼월 설계에 필요한 전반적인 정보를 통합 관리하는 시스템으로, 현재 운영 중인 기존 시스템을 기반으로 전면 개선하는 프로젝트입니다.

| **■ 개선 추진 배경** • 기존 시스템의 노후화로 인한 유지보수 및 기능 확장의 한계 • 제품관리 체계의 비규격화로 외부 시스템(MES 등)과의 연계 불가 • BOM(Bill of Materials) 기반의 정형화된 제품 구성 관리 필요 • 업무 효율화 및 데이터 일관성 확보를 위한 시스템 전반의 현대화 요구 • 현장 실측 업무의 수기 기록 방식에 따른 오류 및 데이터 입력 지연 문제 |
| --- |

## **1.2 프로젝트 범위**

기존 시스템의 주요 기능을 계승하면서, 하기 세 가지 방향으로 개선을 진행합니다.

| **구분** | **내용** |
| --- | --- |
| **기존 계승** | 프로젝트관리, 자재관리, 계약관리, 발주관리, 거래처관리 등 기존 메뉴 구조 유지 및 기능 고도화 |
| **신규 설계** | 제품관리 모듈을 BOM(Bill of Materials) 국제 규격 기반으로 완전 재설계 |
| **신규 추가 (v1.1)** | ⑤ 현장실측 시스템: 안드로이드 앱 + BLE 레이저 측정 장치 연동 디바이스 개발, 현장 실측 데이터 실시간 수집·전송 체계 구축 |

## **1.3 개발 일정**

총 개발기간: 약 8개월 (상세 일정은 4장 WBS 참조)

**Phase 1 (M1~M3): **① 제품관리 시스템 — BOM·자재·공정·제조룰 관리, MES REST API 연동

**Phase 2 (M4~M8): **② 견적 설계 시스템, ③ 발주 관리 시스템, ④ 제조 관리 시스템, **⑤ 현장실측 시스템 (v1.1 추가)**

# **3. 개선 시스템 설계**

## **3.1 시스템 개선 방향**

기존의 메뉴 중심 구조에서 업무 흐름 기반의 5개 서브시스템으로 재편합니다. 각 시스템은 독립적으로 동작하면서 데이터 흐름으로 유기적으로 연결됩니다.

| **영역** | **현행** | **개선 후** | **비고** |
| --- | --- | --- | --- |
| **제품관리** | 비정형 자재 조합, BOM 미정형화 | 원자재·부자재·공정·제조룰 통합 제조 BOM 관리 | Phase 1 |
| **견적 설계** | 수동 도면 해석, 수기 수량 산출 | CAD 웹뷰어 기반 도면 매핑 + BOM 자동 견적 산출 | Phase 2 |
| **발주 관리** | 수동 발주량 계산 | 최적 절단 계산 기반 자동 발주서 산출 | Phase 2 |
| **제조 관리** | GMB 외부 의존, 수동 제작도 작성 | 제작도 자동 산출 + 절단 지시서 + 추가 발주 자동 비교 | Phase 2 |
| **현장실측** | 줄자·수기 기록 → 사무실 재입력 | 안드로이드 앱 + BLE 레이저 측정기 → 실시간 자동 전송 | **Phase 2 (신규)** |
| **외부 연계** | 연계 인터페이스 없음 | MES REST API 연동 (Phase 1), 향후 MES 연동 확장 | Phase 1~ |

## **3.2 서브시스템별 상세 기능 요건**

### **① 제품 관리 시스템**

창호/커튼월 제품을 구성하는 모든 요소(원자재, 부자재, 공정, 제조룰)를 규격화된 제조 BOM으로 통합 관리합니다.

| **기능 모듈** | **상세 요건** |
| --- | --- |
| **자재 관리** | 원자재(프레임 등) 및 부자재(부속 등)의 규격 정보 등록·관리 / 구매 단가 이력 관리 / BOM 연계를 위한 자재 코드 체계 표준화 |
| **공정 관리** | 도장/코팅, 널링, 절단, 가공, 조립 등 공정 유형별 등록 / 공정별 규격 및 제조 단가 관리 / 제조 BOM 구성 시 공정 항목으로 참조 |
| **제조 BOM 관리** | 원자재 + 부자재 + 공정 + 제조룰을 조합한 다단계 제조 BOM 구성 / BOM 계층 구조 트리 뷰 GUI / 버전 관리 및 변경 이력 추적 / MES REST API 연동을 위한 BOM 인터페이스 설계 (Phase 1 핵심) |
| **제품 메타 정보** | 제품 분류 체계 관리 / 제품 코드 부여 및 관리 / 특허 여부 등 제품 관련 메타 정보 등록·조회 |

### **② 견적 설계 시스템**

CAD 도면을 웹에서 직접 열람하고, 도면 기반으로 제품을 매핑하여 BOM 연동 자동 견적 및 실행 예산을 산출합니다.

| **기능 모듈** | **상세 요건** |
| --- | --- |
| **견적 설계 준비** | CAD 도면 업로드 및 웹 뷰어 변환 / 도면 내 개소(위치) 라벨링 및 관리 / 설계자 배정 및 진행 현황 관리 |
| **견적 설계 진행** | 도면 기반 개소별 제품 매핑 (GUI 인터랙션) / 개소별 치수 입력 / 제품-BOM 연동으로 자재 구성 자동 반영 |
| **견적 출력** | 설계 기반 제품 수량 및 치수 산출 / 판매 단가 기준 견적금액 산출 / 견적서 출력 (발주처 제출용) |
| **실행 예산(1차) 산출** | 원가·제조 단가 기준 프로젝트 예산 산출 / 판매 견적 대비 원가율 분석 / 프로젝트 수익성 사전 검토 자료 제공 |

### **③ 발주 관리 시스템**

계약 확정 후 사양을 확정하고, 최적 절단 계산을 통해 원자재 발주량을 자동 산출하여 1차 발주서를 출력합니다.

| **기능 모듈** | **상세 요건** |
| --- | --- |
| **계약 확정 관리** | 견적 기반 계약 처리 및 계약 정보 등록 / 색상, 기타 납품 옵션 등 사양 확정 / 계약 이력 및 상태 관리 |
| **최적 절단 산출** | 원자재 정척 기준 최적 절단 조합 자동 계산 / 손실률 최소화 절단 조합 알고리즘 / 절단 데이터 산출 및 관리 |
| **발주서(1차) 출력** | 최적 절단 조합 기준 자재 발주량 자동 계산 / 거래처별 1차 발주서 출력 / 발주 승인 워크플로우 |

### **④ 제조 관리 시스템**

⑤ 현장실측 시스템에서 전송된 실측 치수를 기반으로 제작도 데이터를 산출하고, 최종 절단 지시 및 1차 발주 대비 추가 발주량을 자동 비교·산출합니다. *(v1.1: 실측 입력 모듈이 ⑤ 현장실측 시스템으로 분리)*

| **기능 모듈** | **상세 요건** |
| --- | --- |
| **실측 데이터 수신** | ⑤ 현장실측 시스템(안드로이드 앱)으로부터 REST API를 통해 실측 데이터 수신 / 개소별 최종 현장 치수 자동 반영 / 수신 데이터 검증 및 이상값 알림 |
| **제작도 데이터 출력** | 제품별 제조용 치수 및 구조 전환 (기존 GMB 기능 대체) / 정형 제품: 제작도 자동 출력 / 비정형 제품: 수동 편집기 제공 / MES 연동을 위한 제작도 데이터 DB 반영 |
| **절단 지시서 출력** | 실측 기반 최적 절단 조합 재계산 / 최종 절단 데이터 산출 / 제조 현장용 절단 지시서 출력 |

## **3.3 ⑤ 현장실측 시스템 (v1.1 신규)**

# **6. 기대 효과**

## **6.1 운영 효율화**

• BOM 기반 자동 자재 소요량 산출로 수작업 오류 감소 및 견적 정확도 향상

• 설계-자재-발주 간 데이터 연결성 확보로 업무 흐름 일원화

• 제품 구성 변경 시 영향도 즉시 파악 가능 (BOM 트레이서빌리티)

## **6.2 시스템 연계성 확보**

• MES 등 외부 시스템과의 표준 API 연동 기반 마련

• BOM 데이터 표준화로 향후 자동화 제조 시스템 연계 가능

• 데이터 일관성 확보로 경영 의사결정 지원 리포팅 기능 확장 가능

## **6.3 중장기 확장성**

• 표준화된 BOM 구조를 기반으로 신제품 등록 및 변형 제품 관리 용이

• 현대화된 아키텍처 기반으로 향후 기능 추가 비용 절감

• 사용자 인터페이스 개선을 통한 직원 업무 만족도 및 생산성 향상

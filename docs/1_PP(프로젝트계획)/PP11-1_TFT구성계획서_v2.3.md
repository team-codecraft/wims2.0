# **1. 사업 개요**

## **1.1 목적**

WIMS(Window Information Management System)는 창호/커튼월 설계에 필요한 모든 정보를 관리하는 종합 시스템이다. 기존 시스템의 노후화 및 비규격화된 제품관리 체계를 개선하고, BOM 기반의 규격화된 제품관리 시스템을 신규 설계하여 MES 등 외부 시스템과의 연계를 확보하는 것을 목적으로 한다.

## **1.2 범위**

본 프로젝트는 5개 서브시스템을 대상으로 하며, 2-Phase 순차 오픈 전략을 적용한다.

- Phase 1 (M1~M3): ① 제품관리 시스템 — 자재·공정·제조 BOM·제품 메타 정보 관리, MES REST API 연동 [v1.2]

- Phase 2 (M4~M8): ② 견적 설계 시스템, ③ 발주 관리 시스템, ④ 제조 관리 시스템, ⑤ 현장실측 시스템 [v1.1 추가]

*[v1.1 변경] ⑤ 현장실측 시스템이 독립 서브시스템으로 신설되었다. 안드로이드 네이티브 앱과 BLE 레이저 측정 장치를 연동하여 현장 실측 데이터를 실시간으로 WIMS 서버에 전송하고, ④ 제조관리 시스템이 이를 수신하여 제작도 산출 및 절단 지시서 출력에 활용한다.*

## **1.3 MES 연동 방식**

REST API 인터페이스 방식을 채택한다. WIMS에서 BOM 데이터를 제공하는 REST API를 개발하고, MES가 해당 API를 호출하여 데이터를 수신하는 구조이다. M1 단계에서 MES 팀과 API 인터페이스 규격(엔드포인트, 요청/응답 스키마, 인증 방식)을 사전 협의하여 확정한다.

*[v1.2 변경] MES 연동 방식이 DB 직접 연동에서 REST API 인터페이스로 전환되었다. API 인증은 JWT 토큰 기반으로 적용하며, MES 전용 서비스 계정을 발급한다. WIMS 내부 DB 구조 변경이 MES에 직접 영향을 주지 않도록 API 레이어에서 데이터 변환·캡슐화를 처리하고, API 버전 관리(v1, v2) 전략을 적용하여 MES 측 호환성을 유지한다.*

# **8. 변경관리 프로세스**

프로젝트 수행 중 발생하는 요구사항 변경, 범위 변경, 일정 변경 등을 체계적으로 관리하기 위한 프로세스를 정의한다.

## **8.1 변경 요청 절차**

- 변경 요청 등록: 변경 요청자가 카카오톡 변경요청 톡방에 변경 사유 및 요청 내용을 등록

- 영향도 분석: PM 및 관련 담당자가 일정·비용·품질 영향도 분석 (2영업일 이내)

- 변경 심의: 발주사 + PM이 변경 승인/반려/보류 결정

- 변경 반영: 승인된 변경 사항을 백로그에 반영, 스프린트 계획 조정

- 변경 검증: 변경 사항 구현 후 테스트 검증, 결과 보고

## **8.2 변경 등급 기준**

- 경미 변경: UI 수정, 문구 변경 등 — PM 승인으로 처리

- 일반 변경: 기능 추가/수정, 비즈니스 로직 변경 — 발주사 실무 담당자 승인

- 중대 변경: 범위 확대, 일정 변경, 아키텍처 변경 — 발주사 대표(유미숙) 승인 필수

*Phase 1 구간(M1~M3)에서의 범위 변경은 1차 오픈 지연 리스크를 감안하여 원칙적으로 동결하며, 불가피한 경우 Gate 3 검토 시점까지 보류 처리한다.*

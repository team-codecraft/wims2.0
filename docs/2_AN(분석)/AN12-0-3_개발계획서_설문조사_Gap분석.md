# 개발계획서 기능요건 vs 설문조사 Gap 분석

**문서코드:** AN12-0-3
**버전:** v1.0
**작성일:** 2026.03.25
**작성자:** 김지광
**근거:** 개발계획서(v1.2) §3.2~3.3, 사전설문조사 결과서(AN11-2 v1.0)

---

## 1. 분석 목적

개발계획서에 정의된 서브시스템별 기능 모듈과 설문조사에서 수집된 요구사항을 대조하여, 설문에서 커버된 영역과 누락된 영역을 식별한다. 누락 영역은 요구사항 정의서(AN12-1) 작성 시 개발계획서 기반으로 보충한다.

---

|-------------------------------|--------------|---------------|----------------|
| **자재 관리** — 원/부자재 규격 등록, 구매 단가 이력, 코드 체계 표준화 | ✅ 부분 커버 | 자재 코드 표준화, 자재 등록 간소화/복사(핵심#9), ~~단가 자동 업데이트~~ (불수용: 외부 시스템 미정의) | 구매 단가 이력 관리 상세, 부자재 분류 기준 |
| **공정 관리** — 도장/코팅, 널링, 절단, 가공, 조립 등록, 공정별 제조 단가 | ❌ 미커버 | — | **전량 보충**: 공정 유형 등록, 공정별 규격·단가, BOM 내 공정 참조 |
| **제조 BOM 관리** — 다단계 BOM 구성, 트리 뷰, 버전 관리, MES 인터페이스 | ⚠️ 간접 커버 | BOM 체계화(업체 요청), 필요수량 자동산출 | **대부분 보충**: BOM 계층 구조 CRUD, 트리 뷰 GUI, 버전 관리, 변경 이력 추적 |
| **제품 메타 정보** — 분류 체계, 제품 코드, 특허 여부 등 | ⚠️ 간접 커버 | 제품 분류 개선 | 제품 코드 부여 규칙, 메타 정보 항목 정의 |

**PM 요약:** 설문은 자재 관리 불편사항 중심이며, BOM·공정·메타정보는 업체 주도 요건으로 개발계획서에서 보충 필요.

## 3. Gap 분석 요약

### 3.1 커버율 총괄

| 서브시스템 | 개발계획서 기능 모듈 수 | 설문 커버 | 부분 커버 | 미커버 | 커버율 |
|-----------|----------------------|----------|----------|--------|--------|
| PM (제품관리) | 4 | 0 | 2 | 2 | 25% |
| ES (견적설계) | 4 | 1 | 3 | 0 | 63% |
| OM (발주관리) | 3 | 0 | 2 | 1 | 33% |
| MF (제조관리) | 4 | 0 | 1 | 3 | 6% |
| FS (현장실측) | 5 | 0 | 0 | 5 | 0% |
| CM (공통) | 7 | 3 | 1 | 3 | 50% |
| **합계** | **27** | **4** | **9** | **14** | **31%** |

> ※ **커버율 산식:** (완전커버 × 1 + 부분커버 × 0.5) / 전체 모듈 수

### 3.2 핵심 Gap 항목 (요구사항 정의서 작성 시 최우선 보충)

| 순위 | Gap 항목 | 서브시스템 | 사유 |
|------|---------|-----------|------|
| 1 | BOM 관리 전체 (계층 구조, 트리 뷰, 버전 관리) | PM | Phase 1 핵심인데 설문에서 상세 미수집 |
| 2 | 공정 관리 전체 | PM | 설문 미커버, BOM 구성 필수 요소 |
| 3 | 현장실측 시스템 전체 (5개 모듈) | FS | 설문 0건, 개발계획서 §3.3 전량 반영 |
| 4 | 제조관리 — 제작도/절단지시서/추가발주 | MF | 설문 2건, GMB 대체 핵심 기능 |
| 5 | 계약 확정 관리 | OM | 설문 미커버, 발주 프로세스 선행 기능 |
| 6 | MES REST API 상세 규격 | CM/PM | 설문 1건, Phase 1 핵심 연동 |
| 7 | 인증/권한 관리 | CM | 설문 미커버, 시스템 기본 요건 |
| 8 | 로깅/감사 추적 | CM | 설문 미커버, 운영 필수 요건 |

## 4. 다음 단계 활용 방안

- **요구사항 정의서(AN12-1) 작성 시:** 설문 커버 항목은 설문 출처로, 미커버 항목은 개발계획서 출처로 기재
- **설문 43건(견적):** 기능 모듈별로 세분화하여 개별 요구사항 ID 부여
- **Gap 항목:** 개발계획서 기능 요건 문장을 요구사항 서식에 맞게 변환
- **비기능 요구사항:** 별도 6단계(비기능 보완)에서 체계적으로 도출

# OM 발주관리 요구사항

> [!abstract]
> 발주관리(Order Management) 서브시스템 기능 요구사항 10건 + 관련 NFR. 계약 처리·최적 절단·거래처별 발주서·승인 워크플로우·널링 지시·발주 현황 추적을 포함한다.

## 1. OM 서브시스템 개요

| 항목 | 내용 |
|------|------|
| 서브시스템 | ③ 발주관리 (Order Management, OM) |
| 스프린트 | S8~S9 (M5~M6) |
| FR 건수 | 10건 |
| NFR 건수 | 1건 (NFR-PF-OM-001) |
| 핵심 가치 | 최적 절단 알고리즘으로 원가 절감 + 거래처 발주 자동화 |
| 주요 사용자 | 발주 담당자, 관리자(승인), 거래처 |
| 주요 접점 | 견적(ES) → 계약 → 발주서 → 제조(MF) 의 중추 |

**주요 기능 영역:**

1. 계약관리 (FR-OM-001~003): 견적 기반 계약·색상/납품 사양 확정·계약 이력
2. 발주관리 (FR-OM-004~010): 최적 절단 조합·각도 공차·거래처별 발주서·승인 워크플로우·널링 지시서·현황 조회
3. 거래처관리 (FR-OM-009): CRUD

## 2. 기능 요구사항 상세 (10건)

#### FR-OM-001 계약 처리 및 계약 정보 등록

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.3 ③ 발주관리 시스템 (OM) — 10건\|FR-OM-001]] |
| **요구사항명** | 견적 기반 계약 처리 및 계약 정보 등록 |
| **분류** | 기능 > 발주관리 > 계약관리 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-ES-016, FR-OM-003, FR-OM-006 |
| **현행 화면 매핑** | 계약 관리 > 신규 계약 |

**상세 설명:**

승인된 견적을 기반으로 고객과 계약을 체결한다. 계약 정보(계약금액, 납기일, 계약자, 조건)를 등록하고, 추적 가능하도록 관리한다.

**입력 데이터:**
- 견적 ID (선택)
- 고객명, 프로젝트명
- 계약금액 (견적과 동일 또는 협상 후 변경)
- 납기일
- 지불 조건 (계약금 30%, 기성금 40%, 기말금 30% 등)
- 특수 조건 (사항사항 등)

**출력 데이터:**
- 계약 번호 (자동 채번: CON-2026-001)
- 계약 상태 (신규, 진행중, 완공, 취소)
- 계약 타임라인

**비즈니스 규칙:**
- 계약금액 ≤ 견적금액 + 5% (초과금 관리자 승인)
- 납기일 최소 조건 (제품별, 설정값)
- 계약자 변경 시 권한 검증 (PM 이상)

**수용 사유:**
발주 → 제조 전 필수 단계. 계약 기록 관리로 분쟁 예방.

#### FR-OM-004 원자재 정척 기준 최적 절단 조합 자동 계산

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.3 ③ 발주관리 시스템 (OM) — 10건\|FR-OM-004]] |
| **요구사항명** | 원자재 정척 기준 최적 절단 조합 자동 계산 |
| **분류** | 기능 > 발주관리 > 발주관리 |
| **난이도** | 상 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2③ |
| **관련 요구사항** | FR-OM-006, FR-MF-006 |
| **현행 화면 매핑** | 발주 > 최적 절단 계산 |

**상세 설명:**

프로필, 유리 등 원자재의 표준 정척(예: 프로필 6미터, 유리 2000x3000mm)을 기준으로, 견적의 모든 자재 요청을 최소 폐기율로 충족하는 절단 조합을 자동으로 계산한다. 이는 원가 최적화의 핵심.

**예시:**
- 필요: 프로필 1200mm × 15개, 1500mm × 12개, 800mm × 8개
- 표준 정척: 6000mm
- 최적 조합: 6000mm 정척 7개 사용 (절단 방법을 자동 결정)
- 결과 폐기율: 약 5% (수작업 대비 20%)

**입력 데이터:**
- 필요 자재 목록 (규격별 수량)
- 표준 정척 마스터 (자재별 표준 길이/면적)
- 절단 공차 허용값

**출력 데이터:**
- 최적 절단 조합:
  - 정척 개수, 절단 패턴 (각 정척에서 뭘 어떻게 자를지)
  - 폐기율 (%)
  - 소요 비용 추정
- 거래처별 발주서 생성 준비

**비즈니스 규칙:**
- NP-hard 문제 (조합 최적화), 동적계획법 또는 휴리스틱 알고리즘 사용
- 다양한 자재(프로필, 유리, 부자재)별로 독립적으로 계산
- 공차: 자재별로 다름 (예: 프로필 ±5mm, 유리 ±2mm)
- 최적화 우선순위: 비용 절감 > 납기 단축

**수용 사유:**
원가 절감의 최대 효과. 수작업 대비 폐기율 70% 감소 가능. 매출 순익률 향상의 가장 큰 기회.

**구현 시 고려사항:**
- 복잡한 알고리즘: 전문 라이브러리(예: OptaPlanner) 고려
- 계산 시간: NFR-PF-OM-001 참조 (1,000개 자재 기준 10초 이내)

#### FR-OM-007 발주 승인 워크플로우

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.3 ③ 발주관리 시스템 (OM) — 10건\|FR-OM-007]] |

| **분류** | 기능 > 발주관리 > 발주관리 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-OM-006, FR-MF-003 |
| **현행 화면 매핑** | 발주 > 승인 워크플로우 |

**상세 설명:**

발주서 작성 후, 관리자 승인을 거쳐 확정된다. 승인 권한은 금액 기준으로 계층화 (과장→부장→임원).

**워크플로우 단계:**
1. 작성자: 발주서 작성 및 제출
2. 과장: 1000만원 이상 승인
3. 부장: 5000만원 이상 승인
4. 임원: 1억원 이상 승인
5. 확정: 발주서 최종 확정 및 거래처 발송

**입력 데이터:**
- 발주서 데이터
- 승인 여부 (승인/반려/수정요청)

**출력 데이터:**
- 워크플로우 상태 (검토중/승인됨/반려됨)
- 타임라인

**비즈니스 규칙:**
- 승인 완료(`status=APPROVED`) 시점에 MF 도메인으로 "PO 확정 이벤트" 를 내부 이벤트 버스(또는 RabbitMQ 권장)로 발행한다. 이벤트 페이로드: `poNumber`, `resolvedBomId[]`, `deliveryDate`. 수신 측은 FR-MF-001 (실측 수신) 이 아닌 **FR-MF-003 제작도 자동 출력** 의 트리거로 동작.

- 프로필 규격별 마킹 위치 및 텍스트

- 마킹 폰트 및 사이즈

**입력 데이터:**
- 프로필 목록, 부호 정보

**출력 데이터:**
- 널링지시서 PDF

#### NFR-PF-OM-001 최적절단 알고리즘 1,000개 자재 기준 10초 이내 완료

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#2.2 성능 (NFR-PF) — 8건\|NFR-PF-OM-001]] |
| **난이도** | 상 |
| **우선순위** | 상 |
| **출처** | AN12-0-4 비기능 보완 |
| **관련 기능 요구사항** | FR-OM-004, FR-MF-006 |

**상세 기술:**

최적 절단 조합 계산(NP-hard 최적화 문제)을 1,000개 자재 기준 최대 10초 내에 완료하여 결과를 사용자에게 제시해야 한다. (AN12-0-4 원본 기준 통일)

**알고리즘 선택:**
- 동적계획법 (DP, 작은 규모 문제에 최적)
- 또는 휴리스틱 (genetic algorithm, simulated annealing 등, 큰 규모 문제)
- 또는 전문 라이브러리 (OptaPlanner, OR-Tools)

**성능 목표:**
- 최대 자재 종류: 50개
- 최대 절단 요청: 1,000개
- 계산 시간: 10초 이내

**최적화:**
- 캐시 활용 (동일한 자재 조합은 이전 결과 재사용)
- 점진적 개선 (초기 해를 빠르게 구한 후, 시간이 남으면 개선)

# ES 견적설계 요구사항

> [!abstract]
> 견적설계(Estimation & Design) 서브시스템 기능 요구사항 21건 + 관련 NFR. CAD 도면 업로드·뷰어·개소 매핑·BOM 자동 구성·견적서 산출·실행예산 분석 등 견적설계 전 과정을 포함한다.

## 1. ES 서브시스템 개요

| 항목 | 내용 |
|------|------|
| 서브시스템 | ② 견적설계 (Estimation & Design, ES) |
| 스프린트 | S6~S8 (M4~M5) |
| FR 건수 | 21건 |
| NFR 건수 | 2건 (NFR-PF-ES-001, NFR-US-ES-001) |
| 핵심 가치 | CAD 도면 기반 자동 BOM 구성 → 견적서·실행예산 산출 자동화 |
| 주요 사용자 | 설계자(견적 담당), PM, 경영진(실행예산 분석) |
| 핵심 요구사항 | FR-ES-008(롤방충망·6명), FR-ES-004(블록 인식·4명), NFR-PF-ES-001(도면 렌더링·4명) |

**주요 기능 영역:**

1. 도면 관리 (FR-ES-001~004): CAD 업로드·렌더링·개소 라벨링·블록 자동 인식
2. 견적 설계 (FR-ES-005~015): 제품 매핑·치수·BOM 자동 구성·롤방충망·미서기·구조검토·복수 일괄 변경
3. 견적서 (FR-ES-016, 019, 021): 비용 합산·PDF 출력·부호 중량 합산 오류 수정
4. 실행예산 (FR-ES-017, 018, 020): 실행율·항목별 비용·총괄 분석

### 1.4 Phase 1 참조 엔티티 · 식별자

> [!info] ES 는 Phase 1 PM 이 확정한 BOM 도메인 모델을 참조한다 (용어사전 §2·§13 기준)
> **수신 엔티티**:
> - `PRODUCT` — 제품 마스터 (series_code, class_l1~l4)
> - `OPTION_GROUP` / `OPTION_VALUE` — 옵션구성 정의 (NUMERIC/ENUM)
> - `PRODUCT_CONFIG` — 옵션구성 스냅샷 (standardBomId 연결)
> - `BOM_RULE` (scopeType=MASTER) — 제품 표준 옵션별규칙
> - `RULE_TEMPLATE` — 규칙 템플릿 (TPL-REINFORCE-SIZE 등 빌트인 6종)
> - `RESOLVED_BOM` — 확정구성표 (RuleEngine 평가 결과 스냅샷)
>
> **핵심 식별자**:
> - `standardBomId` — 제품별 표준 BOM 영속 식별자 (예: `DHS-AE225-D-1`)
> - `standardBomVersion` — BOM 묶음 단일 버전 (sbv{N})
> - `resolvedBomId` — `RBOM-{standardBomId}-sbv{N}-{optionsHash}`
> - `appliedOptions` / `appliedOptionsHash` — 옵션 선택 스냅샷 해시
>
> **견적 단위 오버레이**:
> - `BOM_RULE.scopeType=ESTIMATE` + `estimateId` — 특주·단발성 규칙 (용어사전 §13.4)
>
> **참조 API (MES 외부)**:
> - `GET /api/external/v1/bom/resolved/{resolvedBomId}` — Resolved BOM 조회
> - 상세: [[DE24-1_인터페이스설계서_v2.0]] §4.1

## 2. 기능 요구사항 상세 (21건)

#### FR-ES-001 CAD 도면 업로드 및 웹 뷰어 변환

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-001]] |
| **요구사항명** | CAD 도면 업로드 및 웹 뷰어 변환 |
| **분류** | 기능 > 견적설계 > 도면관리 |
| **난이도** | 상 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2② |
| **관련 요구사항** | FR-ES-002, FR-ES-004, NFR-PF-ES-001 |
| **현행 화면 매핑** | 도면 뷰어 > 도면 선택기 |

**상세 설명:**

사용자가 클라이언트(발주처) 제공 CAD 도면(.dwg, .dxf)을 웹 브라우저에서 업로드하면, 서버가 변환 서비스를 통해 웹 뷰어(SVG 또는 WebGL)로 렌더링할 수 있는 형식으로 변환한다. 변환 완료 후 브라우저에서 줌, 팬, 레이어 표시/숨김, 측정 등 기본 CAD 뷰어 기능을 제공한다.

**입력 데이터:**
- CAD 파일: .dwg, .dxf (최대 50MB)
- 파일 메타정보: 버전, 좌표계, 단위(mm/cm/m)

**출력 데이터:**
- 변환된 도면 바이너리 (SVG 또는 WebGL 포맷)
- 도면 썸네일 이미지
- 레이어 목록 메타정보

**비즈니스 규칙:**
- 변환 타임아웃: 5분 초과 시 실패 처리
- 지원 CAD 버전: AutoCAD 2010 이상
- 변환 실패 시 업로드 재시도 UI 제공
- 변환된 도면은 프로젝트 스토리지에 30일 보관

**수용 사유:**
견적설계의 출발점. 현행 Navisworks 기반 뷰어를 웹 기반으로 전환하여 설치/유지보수 비용 절감, 태블릿 웹 접근성 향상.

**구현 시 고려사항:**
- 외부 라이브러리: Three.js, Babylon.js 또는 CAD 변환 서비스(예: CloudConvert, Aspose)
- BE: Spring MVC 컨트롤러 + 파일 스토리지(S3), 비동기 변환 큐(SQS)
- FE: Vue 3 캔버스 컴포넌트, 반응형 줌/팬 제스처 처리

#### FR-ES-002 도면 내 개소 라벨링 및 관리

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-002]] |
| **요구사항명** | 도면 내 개소 라벨링 및 관리 |
| **분류** | 기능 > 견적설계 > 도면관리 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2② |
| **관련 요구사항** | FR-ES-001, FR-ES-003, FR-ES-005 |
| **현행 화면 매핑** | 도면 뷰어 > 개소 추가/수정 |

**상세 설명:**

도면 렌더링 후 사용자가 도면 상의 창호/도어 설치 위치(개소)에 라벨(A-1, A-2, B-1 등)을 마킹한다. 각 개소에 대해 다음 정보를 관리:
- 라벨 텍스트 및 좌표 (도면 상 픽셀 또는 실제 좌표)
- 개소 타입 (창호/도어/프레임/기타)
- 높이, 폭 등 초기 치수 정보

**입력 데이터:**
- 도면 이미지 좌표 (클릭/드래그)
- 라벨명: 영문+숫자 조합 (예: A-1, Window-01)
- 개소 타입 선택
- 선택적 좌표: 도면의 실제 건축 좌표(mm 단위)

**출력 데이터:**
- 라벨 마크업 JSON (각 개소의 좌표, 라벨, 타입, 치수)
- 개소 목록 표 (도면과 연동)

**비즈니스 규칙:**
- 라벨은 고유해야 함 (중복 금지)
- 라벨 명명 규칙: 영문 1~3자 + 하이픈 + 숫자 (예: A-1, WALL-05)
- 개소 삭제 시 해당 견적 데이터 자동 삭제 여부 확인 후 진행
- 도면 재로드 시 기존 라벨 정보 자동 복원

**수용 사유:**
견적설계의 필수 단계. 수작업으로 도면에 개소를 표시하는 현행 방식을 디지털화.

| **분류** | 기능 > 견적설계 > 프로젝트 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-ES-005, FR-OM-001 |
| **현행 화면 매핑** | 견적 관리 > 설계 할당 |

**상세 설명:**

프로젝트별로 담당 설계자(BOM 매핑, 견적 산출 담당)를 배정하고, 각 프로젝트의 진행 단계를 추적한다:
- 신규 입력 → 개소 라벨링 → 도면 매핑 → BOM 입력 → 견적 검토 → 최종 확정

- 설계자 사용자 ID

**출력 데이터:**
- 설계자별 할당 프로젝트 목록 (진행률 표시)
- 프로젝트별 진행 단계 히스토리
- 타임라인 차트 (Gantt)

**비즈니스 규칙:**
- 한 프로젝트는 하나의 설계자만 할당 가능 (또는 다중 할당 시 역할 구분)
- 상태 전이: 신규 → 설계중 → 검토 → 확정 (역순 이동 가능, 관리자만)
- 상태 변경 시 타임스탬프 및 변경자 자동 기록

**수용 사유:**
팀 협업 관리. 여러 설계자가 동시에 다중 프로젝트를 처리할 때 진행 상황 투명성 제공.

#### FR-ES-004 CAD 블록/외부참조 자동 인식

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-004]] |
| **요구사항명** | CAD 블록/외부참조 자동 인식 |
| **분류** | 기능 > 견적설계 > 도면관리 |
| **난이도** | 상 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **핵심 요구사항** | #2 (4명: 이미희, 김나현, 김수연, 주상원) |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-ES-001, FR-ES-002, FR-ES-005 |
| **현행 화면 매핑** | 도면 뷰어 > 블록 인식 자동화 |

**상세 설명:**

CAD 도면에서 사전 정의된 블록(symbol) 또는 외부참조(xref) 파일의 객체들을 자동으로 인식하고, 각각을 개소 후보로 제시한다. 예를 들어, 도면 라이브러리에서 "창호_1000x1200" 블록이 여러 개 배치되어 있으면, 자동으로 감지하고 개소 리스트를 생성한다.

**입력 데이터:**
- CAD 도면 (변환 완료된 상태)
- 블록 정의 사전: {블록명: 제품명, 기본 높이, 기본 폭}
- 외부참조 문서 경로

**출력 데이터:**
- 인식된 블록 인스턴스 목록 (좌표, 회전, 스케일)
- 제안된 개소 목록 (자동 라벨 생성: A-1, A-2, ...)
- 인식 신뢰도 점수 (%)

**비즈니스 규칙:**
- 블록 사전은 관리자가 사전 등록 (견적설계 > 블록 관리)
- 인식 신뢰도 <80%인 객체는 "검토 필요" 플래그 표시
- 블록 스케일 조정: 도면 설정 좌표계 기반으로 실제 크기 역산
- 중복 인식 방지: 겹침 거리 5mm 이상 떨어진 객체만 별도 개소로 분류

**수용 사유:**
가장 선호되는 요구사항 중 하나. 도면 표준화 시 수작업 라벨링 시간 70% 이상 단축 가능. 현행 수작업 대비 오류 감소.

**구현 시 고려사항:**
- CAD 파일 파싱: Open Design Alliance (ODA) 라이브러리 또는 LibreCAD
- BE: 블록 인식 알고리즘 (좌표 매칭, 회전/스케일 보정)
- FE: 제안 목록 UI, 클릭으로 개소 일괄 추가/필터링 기능

| **출처** | AN11-2(설문), 개발계획서 §3.2② |

| **현행 화면 매핑** | 견적 관리 > 제품 선택 |

**상세 설명:**

라벨링된 각 개소에 대해, `PRODUCT` 엔티티 조회 (series_code·class_l1~l4 로 카탈로그 위치 특정) 를 통해 제품을 선택한다. 사용자는 다음 정보를 기입:
- 개소명 (자동 표시)
- 제품명 (드롭다운 또는 검색)
- 계열 (창호 계열별 BOM 달라짐)
- 개수 (동일 제품의 복수 개소 일괄 선택 가능)

**입력 데이터:**
- 개소 ID
- 제품 선택: 제품명, 계열, 옵션(색상, 유리 등)
- 수량

**출력 데이터:**
- 개소-제품 매핑 테이블
- 매핑 완료율 (%)

**비즈니스 규칙:**
- 제품 선택 필수 (필드 유효성 검사)
- 한 개소에 복수 제품 할당 불가 (1:1 매핑)
- 제품 변경 시 기존 BOM 맵핑 자동 해제

**수용 사유:**
견적 산출의 필수 선행 단계. 도면과 제품 데이터 연결.

#### FR-ES-006 개소별 치수 입력

#### FR-ES-007 제품-BOM 연동 자재 구성 자동 반영

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-007]] |
| **요구사항명** | 제품-BOM 연동 자재 구성 자동 반영 |
| **분류** | 기능 > 견적설계 > 견적설계 |
| **난이도** | 상 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2② |
| **관련 요구사항** | FR-ES-005, FR-ES-006, FR-ES-008~021 |
| **현행 화면 매핑** | 견적 관리 > BOM 자동 구성 |

**상세 설명:**

개소에 할당된 제품과 치수를 기반으로, 자동으로 필요한 자재(원자재, 부자재, 부품), 공정비, 제조비를 계산하고 BOM을 구성한다. 다음 4단계 플로우로 동작한다:

> 1. 선택된 제품의 `standardBomId` 조회
> 2. 사용자가 입력한 옵션값(`appliedOptions`)을 `PRODUCT_CONFIG` 로 스냅샷
> 3. RuleEngine 이 `BOM_RULE` (scopeType=MASTER + scopeType=ESTIMATE 오버레이) 을 평가하여 MBOM 에 변경 적용
> 4. `RESOLVED_BOM` 생성 + `resolvedBomId` 발급 → 견적 라인에 불변 바인딩

예:
- 제품 "미닫이이중창-1000x1200"이 선택되고 높이 1100mm, 폭 950mm로 입력되면
- 자동으로 프로필, 유리, 스페이서, 하드웨어, 작업비 등을 계산하여 자재비 산출

**입력 데이터:**
- 개소 ID, 할당 제품(`standardBomId`), 치수 (H, W)
- 추가 옵션 (`appliedOptions`: 색상, 유리 타입, 하드웨어 등)

**출력 데이터:**
- BOM 아이템 목록:
  - 자재명, 규격, 수량, 단가, 소계
  - 구분: 원자재 | 부자재 | 공정 | 제조비
- `resolvedBomId` (견적 라인 바인딩 키)
- BOM 검증 결과 (오류, 경고)

**비즈니스 규칙:**
- 옵션별규칙은 DE35-1 업무규칙정의서 참조
- 제품 변경 시 기존 Resolved BOM 자동 해제, 신규 `resolvedBomId` 발급
- 치수 변경 시 `appliedOptions` 갱신 후 RuleEngine 재평가 (수동 확인 후 적용)
- 맞춤형 자재(특주) 항목은 `BOM_RULE.scopeType=ESTIMATE` 오버레이로 해당 견적에 한정 추가

**수용 사유:**
견적설계의 핵심 자동화 로직. BOM 수작업 입력 시간 대폭 단축.

**구현 시 고려사항:**
- BE: RuleEngine (빌트인 `RULE_TEMPLATE` + 사용자 정의 `BOM_RULE` 평가)
- `PRODUCT_CONFIG` 스냅샷 보관 (optionsHash 기반 중복 방지)
- 치수별 자재 갯수 산식 (예: 프로필 길이 = (H+W+여유)*2)

- 개소별 높이(H), 폭(W) 치수

- 롤 단가: 높이×폭 비례 (kg당 단가)
- 여러 개소가 동일 규격이면 합산
- 최적 절단 조합: 표준 롤 길이(10m 등) 내에서 낭비 최소 조합 추천

**수용 사유:**
전원 참여 요청 사항. 현행 수작업 산출 → 자동화로 시간 단축, 오류 감소. 매출 원가율 개선.

**구현 시 고려사항:**
- BE: 최적 절단 알고리즘 (NP-hard, 동적계획법 또는 그리디 근사)
- 마스터 데이터: 방충망 규격, 롤 높이, 단가 테이블
- FE: 산출 결과 시뮬레이션 차트 (높이별 롤 개수, 낭비 시각화)

#### FR-ES-016 자재비·시공비·기타비용 합산 및 견적서 산출

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-016]] |
| **요구사항명** | 자재비·시공비·기타비용 합산 및 견적서 산출 |
| **분류** | 기능 > 견적설계 > 견적서 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2② |
| **관련 요구사항** | FR-ES-007, FR-ES-011, FR-ES-017, FR-ES-019 |
| **현행 화면 매핑** | 견적서 > 작성 |

**상세 설명:**

개소별로 산출된 자재비, 시공비(노동비), 기타비용(수송비, 설치비 등)을 합산하여 최종 견적 금액을 계산하고, 견적서 문서를 생성한다. 견적 확정 시점에 모든 개소의 Resolved BOM 이 생성·frozen=TRUE 로 고정된다.

**산식 — 비용 구성:**
- 자재비 = Σ(BOM 자재 단가 × 수량)
- 시공비 = 자재비 × 시공비율(%) [설정값]
- 기타비용 = 자재비 × 기타비율(%) [설정값]
- **소계** = 자재비 + 시공비 + 기타비용
- **부가세(VAT)** = 소계 × 10%
- **총 견적액** = 소계 + 부가세

**입력 데이터:**
- 개소별 BOM (자재, 수량, 단가)
- 시공비율, 기타비율 (프로젝트/회사 기본값)
- 할인율 (선택사항)

**출력 데이터:**
- 견적서 계산 시트:
  - 개소별 항목 상세
  - 자재비/시공비/기타비용 구분
  - 소계, 부가세, 총액
- `resolvedBomId[]` (개소별 1건, OM→MF 바인딩 키)
- `standardBomId` · `standardBomVersion` (버전 추적용)
- 견적서 PDF 문서 (FR-ES-019)

**비즈니스 규칙:**
- 부가세 계산: 소계 기준 10% (설정 변경 가능)
- 할인: 총액의 일괄 할인 또는 항목별 할인
- 반올림: 원 단위 (일반적으로 5원 단위)
- 견적 유효기간: 발주 후 3개월 [설정값]

**수용 사유:**
견적서 작성의 핵심. 정확한 가격 산출로 신뢰성 향상.

**실행율 계산:**
- 원가 구성: 자재비(주요) + 시공비(부)

- **실행율(%) = 실행 원가율 / 견적 원가율 × 100**
  - 100% 이상: 손실 (과원가)
  - 100% 미만: 이익

**입력 데이터:**
- 견적 원가 정보 (FR-ES-016)
- 실행예산 원가 정보 (FR-ES-018)
- 매출액

**출력 데이터:**
- 원가율 (%), 실행율 (%)
- 이익/손실액
- 원가 변동 분석 (자재비 증감, 시공비 변동 등)
- 추이 차트

**비즈니스 규칙:**
- 계산 기준: 소재비(VAT 제외) 기준
- 반올림: 소수점 2자리
- 분석 대상: 완공된 프로젝트만

**수용 사유:**
경영진 의사결정의 핵심 지표. 현행 수작업 오류 → 자동 계산으로 신뢰도 향상.

완성된 견적서를 PDF 형식으로 출력하여 발주처에 제출한다. 회사 로고, 서명란, 법적 문구 포함.

**견적서 포함 내용:**
1. 헤더 (회사명, 로고, 견적 번호, 작성일)
2. 수주자 정보 (회사 정보, 은행 계좌)
3. 발주처 정보 (프로젝트명, 담당자, 주소)
4. 견적 명세 (개소별 상세)
5. 합계 (소재비, 부가세, 총액)
6. 지불 조건, 유효기간, 비고
7. 서명란 (인쇄 후 인증 스탬프)

**입력 데이터:**
- 견적 데이터 (FR-ES-016)
- 템플릿 선택 (표준/간편 등)

**출력 데이터:**
- PDF 파일 (다운로드, 이메일 발송 옵션)
- 인쇄 용지: A4, 장수는 내용에 따라 가변

**비즈니스 규칙:**
- 견적 번호: 자동 채번 (예: EST-2026-001)
- 유효기간: 3개월 [설정값]
- PDF 서명: 사전 설정된 이미지(스탬프) 삽입 가능
- 버전 관리: 견적 수정 시 Rev. 버전 기록 (Rev.0, Rev.1, ...)

**수용 사유:**
발주처 제출 공식 문서. 회사 신뢰성 및 전문성 표현.

#### FR-ES-020 실행예산 총괄 분석 (원가율·수익성 검토)

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-020]] |
| **요구사항명** | 실행예산 총괄 분석 (원가율·수익성 검토) |
| **분류** | 기능 > 견적설계 > 실행예산 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2② |
| **관련 요구사항** | FR-ES-016, FR-ES-017, FR-ES-018 |
| **현행 화면 매핑** | 대시보드 > 실행예산 분석 |

**상세 설명:**

다중 프로젝트의 실행 원가 및 수익성을 대시보드에서 한눈에 분석한다. 프로젝트별, 제품별, 거래처별 원가율, 이익률 추이를 시각화한다.

**분석 내용:**
1. **프로젝트별 분석:**
   - 견적액 vs 실행액 비교
   - 원가율 (%), 이익률 (%)
   - 상태 (완공, 진행중, 손실 위험 등)

2. **추이 분석:**
   - 월별 원가율 추이
   - 제품 카테고리별 수익성
   - TOP 10 손실 프로젝트

3. **거래처 분석:**
   - 거래처별 평균 원가율
   - 거래처별 거래액, 이익액

**입력 데이터:**
- 조회 기간 (월/분기/연도)
- 필터 (프로젝트, 제품, 거래처)

**출력 데이터:**
- 대시보드 차트 (막대, 선, 파이)
- 상세 테이블 (엑셀 다운로드 가능)

**비즈니스 규칙:**
- 원가율 경고: >90%일 경우 주황색, >100%일 경우 빨간색 표시
- 이익률 계산: (매출액 - 원가) / 매출액 × 100
- 분석 대상: 완공된 프로젝트만

**수용 사유:**
경영 의사결정 지원. 수익성 분석으로 가격 책정 전략 수립.

#### FR-ES-021 식별번호 매칭 부호 중량 합산 오류 수정

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.2 ② 견적설계 시스템 (ES) — 21건\|FR-ES-021]] |
| **요구사항명** | 식별번호 매칭 부호 중량 합산 오류 수정 |
| **분류** | 기능 > 견적설계 > 견적설계 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 최상 |
| **핵심 요구사항** | #13 (1명: 김진호) |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-ES-011, FR-ES-007 |
| **현행 화면 매핑** | 견적 관리 > 부호 중량 확인 |

**상세 설명:**

현행 시스템에서 창호부호와 자재(프로필, 유리 등)의 식별번호 매칭 과정에서 중량 합산 오류가 발생했던 문제를 해결한다. 특히, 동일 규격의 복수 개소에서 부호별 중량이 정확히 합산되도록 개선.

**오류 사례:**
- 현행: WD-A-001, WD-A-002, WD-A-003이 모두 1000x1200 규격이면, 각각의 프로필 식별번호가 중복되어 중량이 정확히 합산되지 않음
- 개선: 각 부호별로 고유 식별번호 할당, 합산 시 정확히 계산

**입력 데이터:**
- 창호부호 목록
- 자재 식별번호 매트릭스

**출력 데이터:**
- 부호별 정확한 중량 합계
- 자재별 총 중량

**비즈니스 규칙:**
- 식별번호: 부호 + 자재타입 + 순서 (예: WD-A-001-PF-001 = WD-A-001의 프로필 첫 번째)
- 합산 검증: 부호별 자재 개수 확인 후 합산

**수용 사유:**
현행 시스템 버그 수정. 정확한 자재 발주를 위한 필수 수정.

#### NFR-PF-ES-001 대용량 CAD 도면(50MB) 10초 이내 렌더링

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#2.2 성능 (NFR-PF) — 8건\|NFR-PF-ES-001]] |
| **난이도** | 상 |
| **우선순위** | 최상 |
| **핵심 요구사항** | #3 (4명: 이미희, 김수연, 주상원, 김진호) |
| **출처** | AN11-2(설문) |
| **관련 기능 요구사항** | FR-ES-001 |

**상세 기술:**

50MB 규모의 CAD 도면을 업로드한 후, 웹 뷰어에서 초기 렌더링을 완료하는 데 10초 이내 소요되어야 한다.

**성능 목표:**
- 파일 업로드: 5초 이내 (네트워크 대역폭 10Mbps 기준)
- 서버 변환: 3초 이내
- 웹 뷰어 초기 렌더링: 2초 이내
- 합계: 10초 이내

**최적화 전략:**
1. 도면 변환 시 선택적 층(layer) 로딩 (모든 층을 한 번에 렌더링하지 않음)
2. 웹 뷰어 지연 로딩 (viewport 내 객체만 렌더)
3. 서버에서 캐시 활용 (동일 도면 재업로드 시 캐시 사용)
4. CDN을 통한 변환 결과 배포

**검증 방법:**
- 실제 50MB CAD 파일로 테스트
- Chrome DevTools Network/Performance 탭에서 측정
- 다양한 네트워크 환경에서 테스트 (4G, Wi-Fi)

#### NFR-US-ES-001 견적 조회 반응형 UI (태블릿 이상, 일부수용)

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#2.3 사용성 (NFR-US) — 8건\|NFR-US-ES-001]] |
| **난이도** | 중 |
| **수용여부** | **일부수용** |
| **우선순위** | 하 |
| **출처** | AN11-2(설문) |
| **관련 기능 요구사항** | FR-ES-016, **FR-CM-006 (Phase 1에서 정의: 태블릿·데스크톱 반응형 웹 UI 지원)** |

**일부수용 사유:**

모바일 브라우저에서 견적서를 조회할 수 있는 기능을 요청했으나, 스마트폰 화면 사이즈는 WIMS 시스템의 복잡한 UI 특성상 지원하지 않기로 결정했다. 태블릿(768px 이상) 해상도에서 견적 조회를 지원하며, 이는 Phase 1에서 정의한 **FR-CM-006 (태블릿·데스크톱 반응형 웹 UI 지원, 768px 이상)**에 준하여 구현된다.

**지원 범위 (768px 이상):**
- 견적서 PDF 다운로드 (태블릿 브라우저에서 열기 가능)
- 견적 요약 정보 조회 (태블릿 반응형 테이블)

**제외 범위:**
- 스마트폰(768px 미만) 화면 사이즈 대응
- 견적 수정/작성 (조회만 가능)
- 오프라인 조회

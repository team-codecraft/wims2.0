# NFR 공통·RTM

> [!abstract]
> Phase 2 비기능 요구사항(NFR) 카테고리 개요 + RTM(요구사항 추적 매트릭스). 개별 NFR 상세는 서브시스템별 섹션 파일(ES·OM·MF·FS)에 연결된 FR 과 함께 분산 배치했다. 본 섹션은 카테고리 총람·서브시스템 매트릭스·출처별 추적만 수록한다.

## 1. NFR 카테고리 개요

Phase 2 신규·추가 NFR 은 15건. 모두 서브시스템별 섹션에 분산 배치되어 있으며, 본 섹션은 분산 원칙을 명시한다.

| 카테고리 | 코드 | Phase 2 건수 | 분산 위치 |
|---------|------|------------|----------|
| 보안 | SC | 2 | 모두 FS ([[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.1 보안 (SC) — 2건\|§3.1]]) |
| 성능 | PF | 4 | ES 1, OM 1, FS 3 ([[AN12-1_요구사항정의서_Phase2/sections/ES_견적설계#NFR-PF-ES-001\|ES]] · [[AN12-1_요구사항정의서_Phase2/sections/OM_발주관리#NFR-PF-OM-001\|OM]] · [[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.2 성능 (PF) — 3건\|FS]]) |
| 사용성 | US | 3 | ES 1, FS 2 ([[AN12-1_요구사항정의서_Phase2/sections/ES_견적설계#NFR-US-ES-001\|ES]] · [[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.3 사용성 (US) — 2건\|FS]]) |
| 신뢰성 | RL | 4 | 모두 FS ([[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.4 신뢰성 (RL) — 4건\|§3.4]]) |
| 인터페이스 | IF | 2 | 모두 FS ([[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.5 인터페이스 (IF) — 2건\|§3.5]]) |
| 이식성 | PT | 2 | 모두 FS ([[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#3.6 이식성 (PT) — 2건\|§3.6]]) |
| **합계** | | **15** | ES 2 · OM 1 · FS 12 |

> [!note] 분산 배치 근거
> Phase 2 NFR 은 대부분 특정 FR 에 종속(예: NFR-PF-ES-001 은 FR-ES-001 전용, NFR-IF-FS-001 은 FR-FS-001/002 의 BLE GATT 인터페이스 정의). 서브시스템 섹션 안에서 함께 읽는 편이 설계·구현 단계에서 더 효과적. 서브시스템 무관 전역 NFR 은 Phase 2 v1.0-r1 시점 별도 정의된 항목이 없으며, 향후 도출 시 본 섹션 §2 로 이관한다.

## 2. Phase 2 공통(전역) NFR

_현재 본 섹션에 배치된 전역 공통 NFR 은 없음._

향후 개방이슈:
- Phase 2 전체 공통 가용성(예: 월 99.5% 이상) — 개발계획서 §4 운영 목표와 조율 후 정의
- 전체 로깅·감사 추적 공통 NFR — Phase 1 NFR-SC 계열과 통합 검토
- 재해복구(DR) RTO/RPO — 인프라 팀 확정 후 정의

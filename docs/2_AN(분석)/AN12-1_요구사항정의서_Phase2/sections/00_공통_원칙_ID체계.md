# 공통 원칙·ID 체계

> [!abstract]
> Phase 2 요구사항 정의서의 공통 섹션. §1 개요(목적·범위·참조·용어·이해관계자), §2 총괄 통계·우선순위 분포·핵심 요구사항, ID 체계 정의를 포함한다. 서브시스템별 상세(FR-ES/OM/MF/FS)는 각각 분리된 섹션 파일 참조.

## 1. 개요

### 1.1 목적

본 문서는 WIMS 2.0 Phase 2 (M4~M8) 구현 범위의 기능 요구사항(FR) 53건과 비기능 요구사항(NFR)을 특허청 SW 개발방법론 AN12-1 형식으로 정의한다. 분석 단계에서 도출된 요구사항을 설계·구현 단계에서 활용 가능하도록 구체화한다.

### 1.2 범위

**대상 서브시스템 (4개):**

- ② 견적설계 시스템 (Estimation & Design, ES)
- ③ 발주관리 시스템 (Order Management, OM)
- ④ 제조관리 시스템 (Manufacturing, MF)
- ⑤ 현장실측 시스템 (Field Survey, FS)

**적용 기간:** Phase 2 (M4~M8) = 스프린트 S6~S11 (약 24주)

**제외 범위:**

- Phase 1 (① 제품관리 시스템) — 별도 [[AN12-1_요구사항정의서_Phase1_v1.1]] 참조
- Phase 3 이후 요구사항

### 1.4 용어 정의

| 용어 | 정의 |
|------|------|
| **개소** | 도면 상의 창호 설치 위치. 건물 평면도에서 A-1, B-2 등으로 표시되는 각 창호/도어 설치점 |
| **CAD 도면** | AutoCAD .dwg, .dxf 형식의 건축 설계도. 개소 정보를 포함한 2D/3D 도면 |
| **BOM (Bill of Materials)** | 제조BOM. 각 제품 규격별 필요한 원자재(프로필, 유리), 부자재(하드웨어), 공정비, 제조룰(가공비, 조립비) 포함 |
| **롤방충망** | 창호에 설치되는 방충망. 롤 형태로 보관하며, 규격(높이×폭)별 중량을 산출하는 기능 |
| **미서기** | 미닫이 창호의 일종. 보강재(하드웨어) 적용 규칙이 있어 자동화 로직 필요 |
| **트랜섬** | 창호의 가로 프레임(수평재). 절단 사이즈가 다른 부재와 연관 |
| **실행예산** | 제조사 내부용 예산. 원가율, 수익성 검토용으로 자재비·시공비·기타비용 합산 |
| **절단 최적화** | 원자재 낭비를 최소화하는 절단 조합 선택. NP-hard 계열 알고리즘 |

> [!note]
> BOM 도메인 상세 용어는 [[WIMS_용어사전_BOM_v1.4]] 참조. Phase 2 신규 용어(BLE, GATT, resolvedBomId 등)는 각 서브시스템 섹션에서 다룬다.

### 1.6 요구사항 ID 체계

Phase 2 요구사항 ID 는 [[AN12-0_요구사항_ID체계_및_분류기준]] 규칙을 따른다.

| 구분 | 형식 | 예시 |
|------|------|------|
| 기능 요구사항 | `FR-{서브시스템}-{3자리 일련번호}` | FR-ES-001, FR-OM-004, FR-MF-006, FR-FS-001 |
| 비기능 요구사항 | `NFR-{카테고리}-{서브시스템}-{3자리 일련번호}` | NFR-PF-ES-001, NFR-SC-FS-001 |
| NFR 카테고리 | SC(보안) / PF(성능) / US(사용성) / RL(신뢰성) / IF(인터페이스) / PT(이식성) | — |
| 서브시스템 코드 | ES / OM / MF / FS (Phase 2) — PM / CM (Phase 1) | — |

> [!info] 서브시스템별 분산 원칙
> Phase 2 는 4개 독립 서브시스템(ES·OM·MF·FS)으로 구성되며 각각 도메인·개발 일정이 분리된다. 본 정의서는 서브시스템별 섹션 파일로 FR 상세를 분산하고, NFR 도 연결된 FR 이 속한 서브시스템으로 함께 배치한다. 전체 RTM 은 [[AN12-1_요구사항정의서_Phase2/sections/NFR_공통|NFR_공통]] 에 집약.

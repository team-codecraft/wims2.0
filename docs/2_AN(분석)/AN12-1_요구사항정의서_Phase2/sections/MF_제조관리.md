# MF 제조관리 요구사항

> [!abstract]
> 제조관리(Manufacturing) 서브시스템 기능 요구사항 11건 (FR-MF-011 v1.1 신설 포함). 현장실측 데이터 수신·제작도 자동 출력·MES 연동·실측 기반 절단 재계산·최종 절단 데이터 산출·1차↔최종 발주 비교·공정 단계별 진행 현황 추적을 포함한다.

## 1. MF 서브시스템 개요

| 항목 | 내용 |
|------|------|
| 서브시스템 | ④ 제조관리 (Manufacturing, MF) |
| 스프린트 | S10~S11 (M7~M8) |
| FR 건수 | 11건 (v1.1: FR-MF-011 신설) |
| NFR 건수 | 0건 (FS·OM 쪽에 연동 NFR 배치: NFR-IF-FS-002 MES REST API, NFR-PF-OM-001 절단 알고리즘) |
| 핵심 가치 | 실측 데이터 기반 재계산으로 폐기율 최소화, MES 연동으로 제조 현장 직결 |
| 주요 사용자 | 제조 담당자, 생산 계획 담당, MES 협력 팀 |
| 주요 접점 | FS(실측 수신) → 제작도·MES → OM(발주 비교·추가 발주) |

**주요 기능 영역:**

1. 실측연동 (FR-MF-001~002): REST API 수신·데이터 검증
2. 제작도 (FR-MF-003~004): 정형 제품 자동 출력·비정형 수동 편집기
3. MES연동 (FR-MF-005): 제작도 데이터 반영
4. 절단관리 (FR-MF-006~008): 실측 기반 재계산·최종 절단 데이터·작업자 지시서
5. 발주비교 (FR-MF-009~010): 1차 vs 최종 비교·추가 발주 처리
6. 공정추적 (FR-MF-011): 공정 단계별 진행 현황 추적 (v1.1 신설)

## 2. 기능 요구사항 상세 (11건)

#### FR-MF-001 현장실측 데이터 수신 (REST API)

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.4 ④ 제조관리 시스템 (MF) — 10건\|FR-MF-001]] |
| **요구사항명** | 실측 데이터 수신 (REST API) |
| **분류** | 기능 > 제조관리 > 실측연동 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2④ |
| **관련 요구사항** | FR-MF-002, FR-MF-006, FR-FS-005 |
| **현행 화면 매핑** | 제조 관리 > 실측 데이터 수신 |

**상세 설명:**

현장에서 수집된 측정 데이터(높이, 폭 등의 실제 치수)를 REST API를 통해 수신하고, 제조 시스템에 반영한다. FS FR-FS-005 과 동일한 이벤트형 페이로드·엔드포인트를 사용하여 단일 접점을 유지한다.

**API 스펙:**
- Endpoint: POST /api/v1/field-survey/measurements (FS 와 동일, 단일 접점)
- 기존 `POST /api/v1/manufacturing/field-survey/receive` 는 삭제 또는 alias 처리
- 요청 본문 (이벤트형, FS FR-FS-005 스키마 통일):
  ```json
  {
    "projectId": "PRJ001",
    "areaId": "A-1",
    "measurements": [
      {"type": "height", "value": 2850.5, "unit": "mm", "timestamp": "2026-04-03T14:30:00Z", "photoUrls": ["url1"], "memo": "문틀 손상"},
      {"type": "width", "value": 1800.0, "unit": "mm", "timestamp": "2026-04-03T14:30:00Z", "photoUrls": [], "memo": ""}
    ]
  }
  ```
- 응답: 수신 확인 (200 OK, message_id)

**입력 데이터:**
- 실측 데이터 (JSON)

**출력 데이터:**
- 수신 확인 응답
- 내부 DB 저장

**비즈니스 규칙:**
- API 호출 인증: JWT 토큰
- 데이터 검증: 필수 필드 확인 후 저장
- 중복 방지: 같은 location/timestamp 데이터는 한 번만 처리
- 응답 시간: 2초 이내

#### FR-MF-002 수신 데이터 검증 및 이상값 알림

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.4 ④ 제조관리 시스템 (MF) — 10건\|FR-MF-002]] |
| **요구사항명** | 수신 데이터 검증 및 이상값 알림 |
| **분류** | 기능 > 제조관리 > 실측연동 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문) |
| **관련 요구사항** | FR-MF-001 |
| **현행 화면 매핑** | 제조 관리 > 데이터 검증 |

**상세 설명:**

수신된 실측 데이터의 유효성을 검증한다. 예상 범위를 벗어난 데이터는 경고 플래그를 붙이고 관리자에게 알림한다.

**검증 규칙:**
1. 서버 2차 검증 — 도면 기준값 ±50mm (엄격, 범위 초과 시 제작 진입 차단):
   - 높이: 설계 높이 ± 50mm 범위 내
   - 폭: 설계 폭 ± 50mm 범위 내
   - 범위 초과 시 "오류" 플래그 + 제작 진입 차단

2. 논리 검증:
   - 높이 > 0, 폭 > 0 (음수 불가)
   - 높이/폭 비율 검사 (불합리한 비율 감지)

3. 중복 검증:
   - 같은 location에서 짧은 시간 내 재전송 감지

**입력 데이터:**
- 수신된 실측 데이터

**출력 데이터:**
- 검증 결과 (OK, 경고, 오류)
- 알림 메시지 (관리자에게 발송)
- 데이터 태그 (검증 상태 기록)

**비즈니스 규칙:**
- 경고 데이터도 DB에 저장 (기록 목적), 하지만 제작도 생성 시 제외
- 오류 데이터: 재측정 요청
- 본 FR 은 2단계 검증 체계의 2차(서버·엄격) 검증 — 1차 현장·모바일 검증은 [[AN12-1_요구사항정의서_Phase2/sections/FS_현장실측#FR-FS-002 실측 데이터 입력 및 유효성 검증|FR-FS-002]] 참조

#### FR-MF-003 정형 제품 제작도 자동 출력

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.4 ④ 제조관리 시스템 (MF) — 10건\|FR-MF-003]] |
| **요구사항명** | 정형 제품 제작도 자동 출력 |
| **분류** | 기능 > 제조관리 > 제작도 |
| **난이도** | 상 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2④ |
| **관련 요구사항** | FR-MF-001, FR-MF-005 |

**상세 설명:**

표준 규격의 제품에 대해, 실측 치수를 기반으로 제작도(제조용 상세도면)를 자동으로 생성한다. 제작도에는 각 부재의 규격, 조립 방법, 주요 치수 등을 포함한다.

**제작도 포함 내용:**
- 조립도 (전체 구성)
- 부재별 상세도 (프로필, 유리, 하드웨어)
- 주요 치수 (높이, 폭, 프로필 절단 길이 등)
- 조립 순서 및 주의사항

**입력 데이터:**
- 제품 ID, 실측 치수
- 제작도 템플릿 (제품별)

**출력 데이터:**
- 제작도 PDF (여러 페이지)
- CAD 데이터 (DWG, 향후 확장)

#### FR-MF-005 MES 연동 제작도 데이터 반영

| 항목 | 내용 |
|------|------|
| **요구사항 ID** | [[AN12-1_요구사항목록_v1.5#1.4 ④ 제조관리 시스템 (MF) — 10건\|FR-MF-005]] |
| **요구사항명** | MES 연동 제작도 데이터 반영 |
| **분류** | 기능 > 제조관리 > MES연동 |
| **난이도** | 중 |
| **수용여부** | 수용 |
| **우선순위** | 상 |
| **출처** | AN11-2(설문), 개발계획서 §3.2④ |
| **관련 요구사항** | FR-MF-003 |

**상세 설명:**

WIMS와 MES(제조실행시스템) 간의 연동을 하이브리드 방식으로 구성한다.

> **하이브리드 방향**:
> - **BOM 조회 (MES→WIMS, pull)**: MES 가 WIMS 의 `GET /api/external/v1/bom/resolved/{resolvedBomId}` 를 호출 (DE24-1 §4.1).
> - **제작도·생산실적 송신 (MF→MES, push)**: MF 가 제작도 데이터 확정 시 MES REST API 로 전송. 생산실적 데이터도 MF 가 MES 로 push (향후 양방향 확장).

**연동 정보:**
- 제작도 ID, 버전
- 제품 규격, BOM
- 공정 정보 (가공 순서, 소요 시간)

**구현:**
- REST API 기반 (MES 팀과 협의, DE24-1 §4.1 Resolved BOM API 준거)

| **난이도** | 중 |

| **관련 요구사항** | FR-MF-009, FR-OM-007 |

**상세 설명:**

1차 발주 후 추가 발주가 필요한 경우, 추가 발주서(2차, 3차 등)를 생성하고 승인 워크플로우를 거친다.

**처리 과정:**
1. 추가 발주 필요 판단 (FR-MF-009)
2. 추가 발주서 자동 생성 (2차 발주서로 표시)
3. 승인 워크플로우 (FR-OM-007)
4. 거래처에 발송

**비즈니스 규칙:**

# AN12-1 요구사항 정의서 Phase 2 v1.0-r1

> [!abstract] v1.0-r1 분산 구조 재편 (2026-04-23)
> - **얇은 인덱스 허브 + sections/ 분산**: 본 파일은 메타·요약·링크만 유지. FR 53 + NFR 14 상세는 서브시스템별 섹션 파일에 분산.
> - **섹션 6개**: 00 공통 원칙·ID 체계 / ES 견적설계 / OM 발주관리 / MF 제조관리 / FS 현장실측 / NFR 공통·RTM
> - **FR 상세 보존**: 기능 요구사항 53건은 원문 그대로 이전. NFR 은 연결된 FR 이 속한 서브시스템 섹션에 함께 배치 (ES 2·OM 1·FS 12, MF 는 OM/FS NFR 참조).
> - **Phase 2 설계 착수 전 재편**: 네이밍·추적 비용이 최저인 시점에 Phase 1 / Phase 2 동일한 분산 원칙을 맞춰 향후 편집·Obsidian 그래프 탐색 효율을 높인다.

## 6. v1.0-r1 개정 핵심 포인트

### 6.1 분산 구조 재편

- **단일 2,802줄 파일 → 인덱스 허브(~120줄) + 6 섹션** 으로 분리
- Obsidian wikilink·그래프뷰로 섹션 간 탐색. 각 섹션은 독립 편집 가능
- Phase 1 ([[AN12-1_요구사항정의서_Phase1_v1.1]]) 와 동일한 분산 원칙 적용 준비 — AN12-1 Phase 1·Phase 2 일관성 확보

### 6.2 FR 원본 보존 원칙

- 기능 요구사항 53건 상세(항목 표·상세 설명·입력/출력·비즈니스 규칙·수용 사유·구현 고려사항) 는 **원문 그대로** 해당 서브시스템 섹션으로 이전
- 버그 수정이 아닌 재구성 목적 — 요구사항 내용·수용 여부·우선순위 변경 없음

### 6.3 NFR 배치 원칙

- 서브시스템 종속 NFR → 해당 서브시스템 섹션에 함께 배치 (ES 2·OM 1·FS 12, FS 의 IF-002/PF-002 는 MF 와도 연결)
- MF 는 전용 NFR 없음 — OM/FS NFR 을 참조 형태로 연결 (개방이슈: MES 연동 전용 NFR 추후 정의)
- 전역 공통 NFR 은 v1.0-r1 시점 없음. 향후 가용성·DR·감사 추적 NFR 도출 시 NFR_공통 §2 로 이관

### 6.4 RTM 권위 출처

- 본 정의서 RTM 은 출처·핵심·카테고리 분포 요약만 담당
- 상세 FR ↔ 화면/DB/테스트 케이스 매핑은 [[AN14-1_요구사항추적표_v1.2]] 단일 권위 출처

### 6.5 Phase 2 설계 단계 준비

- 각 서브시스템 섹션의 "관련 화면·설계 매핑" callout 은 Phase 2 DE22-1 편·DE24-1 연동 시 업데이트 포인트 명시
- ES/OM 은 Phase 1 PM 도메인(resolvedBomId·BOM 옵션 구성) 과의 접점 기입 — [[DE22-1_화면설계서_v1.6]] SCR-PM-013B 확정구성표 참조

## 1. As-Is 현행 업무 프로세스

### 1.1 개요

현행 WIMS 제품관리는 자재관리(원자재/부자재/구성원자재)와 제품관리(판매제품/순번)로 구성된다. 자재 등록 만족도 1.33점(5점 만점)으로 가장 낮은 영역이며, BOM 체계가 부재하고 MES 연동 기능이 없는 상태이다.

### 1.2 현행 업무 흐름도

```mermaid
flowchart TD
    Start([자재/제품 관리 시작])

    Start --> A1[원자재 수동 등록<br/>자재ID/이름/규격/중량 입력]
    A1 --> A2{자재코드<br/>중복 확인?}
    A2 -->|중복체크 없음| A3[중복 자재 등록 가능<br/>⚠️ 데이터 불일치 발생]
    A2 -->|정상| A4[원자재 등록 완료]
    A3 --> A4

    A4 --> B1[구성원자재 그룹 등록<br/>수동으로 자재 조합]
    B1 --> B2[속성설정 수동 입력]

    B2 --> C1[판매제품 등록<br/>분류/이름/등급/재질/단열 설정]
    C1 --> C2[제품 구성요소 수동 매핑<br/>부품설정/허용오차/조건설정]
    C2 --> C3[제작/시공비 수동 설정]

    C3 --> End([종료])
```

## 2. To-Be 목표 업무 프로세스

### 2.1 개요

WIMS 2.0 제품관리 시스템은 자재코드 표준화, 다단계 제조 BOM 체계, MES REST API 실시간 연동을 핵심으로 한다. 자재 등록 간소화(필수항목 우선 입력, 기존 자재 복사)로 사용편의성을 개선하고, BOM 트리뷰 GUI로 제품-자재 계층 구조를 시각적으로 관리한다.

### 2.2 목표 업무 흐름도

#### 2.2.0 마스터 데이터 사전 준비

BOM 구성을 시작하기 전에 자재 마스터와 공정 마스터가 등록되어 있어야 한다. 이 두 가지는 제품 등록과 병행하여 점진적으로 추가할 수 있지만, EBOM 확정 전까지 자재가, MBOM 편집 전까지 공정이 등록되어야 한다.

```mermaid
flowchart TD
    Start([마스터 데이터 준비 시작])

    Start --> P1[프로젝트 등록/선택<br/>FR-PM-017]

    P1 --> A1[자재 등록<br/>표준 코드체계 적용 FR-PM-004]
    A1 --> A2{자재코드<br/>유일성 검증}
    A2 -->|중복| A3[등록 거부 + 기존 자재 안내]
    A2 -->|통과| A4{자재 유형}
    A3 --> A1

    A4 -->|원자재| A5[원자재 규격 등록<br/>FR-PM-001]
    A4 -->|부자재| A6[부자재 규격 등록<br/>FR-PM-002]

    A5 --> A7{등록 방식}
    A6 --> A7
    A7 -->|신규| A8[개별 등록 폼<br/>필수항목 우선 입력]
    A7 -->|복사| A9[기존 자재 복사 후 수정<br/>FR-PM-005]
    A8 --> A10[자재 마스터 DB 저장]
    A9 --> A10

    A10 --> C1[공정 유형 등록<br/>절단/용접/코팅/조립 등 FR-PM-008<br/>화면: SCR-PM-007 기본정보 탭]
    C1 --> C2[공정별 규격 및 제조단가 설정<br/>FR-PM-009<br/>화면: SCR-PM-007 규격·단가 탭]

    C2 --> End([마스터 데이터 준비 완료<br/>→ BOM 구성 진입 가능])
```

#### 2.2.1 BOM 구성 업무 흐름도 (STEP 1~8)

새로운 제품의 BOM을 구성하여 MES에 전달하기까지의 전체 업무 순서이다. 각 단계의 상세 내용은 BOM 구성에 대한 고찰(DE35-1 부록 D) 7장을 참조한다.

```mermaid
flowchart TD
    Start([BOM 구성 시작])

    %% STEP 1: 제품 기본정보 등록
    Start --> S1["<b>STEP 1. 제품 기본정보 등록</b><br/>제품 분류체계 선택, 코드 자동 생성<br/>제품 메타정보 입력<br/>FR-PM-014, 015, 016<br/><i>담당: 설계팀/영업팀 · 화면: SCR-PM-011</i>"]

    %% STEP 2: EBOM 등록
    S1 --> S2["<b>STEP 2. EBOM 등록 — 자재 구성</b><br/>기능군(L1) 생성 → 부품 배치(L2~3)<br/>설계 수량 입력 → 대체부품 등록<br/>FR-PM-010, 011<br/><i>담당: 설계팀 · 화면: SCR-PM-013 (EBOM 모드)</i>"]

    %% STEP 3: EBOM 확정
    S2 --> S3{"<b>STEP 3. EBOM 확정</b><br/>검토 → 승인<br/>FR-PM-012<br/><i>화면: SCR-PM-014</i>"}
    S3 -->|승인 거부| S2
    S3 -->|승인| S3a["EBOM 상태: APPROVED<br/>MBOM 초안 자동 생성"]

    %% STEP 4 & 5 병행
    S3a --> S4["<b>STEP 4. MBOM 편집 — 공정 구성</b><br/>조립체 재구성 → 공정코드 배정<br/>작업순서 → 로스율 → 실소요량<br/>EBOM↔MBOM 매핑 확인<br/>FR-PM-010, 011, 006<br/><i>담당: 생산팀 · 화면: SCR-PM-013 (MBOM 모드)</i>"]
    S3a --> S5["<b>STEP 5. 옵션 그룹/값 정의</b><br/>6개 옵션 차원 정의<br/>옵션 값 등록, 제약 조건 설정<br/><i>담당: 설계팀 · 화면: SCR-PM-013B (옵션 그룹)</i>"]

    %% STEP 6: BOM Rule (STEP 4+5 완료 후)
    S4 --> S6
    S5 --> S6["<b>STEP 6. BOM Rule 정의</b><br/>조건식 + 액션 등록<br/>우선순위 설정, 시뮬레이션 검증<br/><i>담당: 설계팀 · 화면: SCR-PM-013B (규칙 관리)</i>"]

    %% STEP 7: BOM 해석
    S6 --> S7["<b>STEP 7. 옵션 구성 생성 및 BOM 해석</b><br/>옵션 조합 선택 → 제약 검증<br/>Base BOM + Rule 적용 → Resolved BOM 생성<br/>부품 누락/수량 정합성 검증<br/>FR-PM-013<br/><i>담당: 영업팀/설계팀 · 화면: SCR-PM-013B (구성 목록)</i>"]
    S7 --> S7a{검증 통과?}
    S7a -->|실패| S7b[Rule 수정 또는<br/>Base BOM 보완]
    S7b --> S6
    S7a -->|통과| S7c["Config 상태: RESOLVED"]

    %% STEP 8: MES 전달
    S7c --> S8["<b>STEP 8. 확정 구성표 검증 및 MES 전달</b><br/>Resolved BOM 검토 (변경 하이라이트)<br/>소요량 요약 확인 → MES 전달 승인<br/>FR-PM-013<br/><i>담당: 생산팀 리더 · 화면: SCR-PM-013B (확정 구성표)</i>"]
    S8 --> S8a["Config 상태: RELEASED<br/>MES REST API로 확정 구성표 노출"]

    S8a --> End([BOM 구성 완료<br/>MES 조회 가능])

    %% 스타일
    style S1 fill:#E8F0FE,stroke:#1F4E79
    style S2 fill:#E8F0FE,stroke:#1F4E79
    style S3 fill:#FFF3E0,stroke:#E65100
    style S3a fill:#E8F5E9,stroke:#2E7D32
    style S4 fill:#F3E5F5,stroke:#6A1B9A
    style S5 fill:#F3E5F5,stroke:#6A1B9A
    style S6 fill:#E8F0FE,stroke:#1F4E79
    style S7 fill:#E8F0FE,stroke:#1F4E79
    style S7c fill:#E8F5E9,stroke:#2E7D32
    style S8 fill:#FFF3E0,stroke:#E65100
    style S8a fill:#E8F5E9,stroke:#2E7D32
```

> **병행 가능 구간:** STEP 4(MBOM 편집)와 STEP 5(옵션 그룹/값 정의)는 서로 독립적이므로 병행 작업이 가능하다. 다만 STEP 6(BOM Rule 정의)은 MBOM 노드와 옵션 값을 모두 참조하므로, STEP 4와 STEP 5가 모두 완료된 후 진행해야 한다.

> **변경 시 재진입 지점:** BOM Rule만 변경된 경우 STEP 7(재해석)부터 재진행. EBOM 구조가 변경된 경우 STEP 2부터 새 버전으로 재진행.

### 2.2.2 구매 단가 이력 관리 (FR-PM-003) — 별도 업무 흐름

자재 마스터 등록/수정과는 독립적인 흐름으로, 구매·발주 시점에 거래처로부터 확정된 단가를 기록·관리한다.

```mermaid
flowchart TD
    S([구매/발주 단가 관리 시작])

    S --> Q0{자재-거래처<br/>매핑 존재?}
    Q0 -->|없음| Q0a[자재에 거래처 추가<br/>복수 거래처 매핑 가능]
    Q0 -->|있음| Q1
    Q0a --> Q1[거래처별 구매 단가 입력<br/>자재코드, 거래처, 단가, 적용시작일]

    Q1 --> Q2{해당 거래처에<br/>기존 유효 단가 존재?}
    Q2 -->|있음| Q3[기존 단가 적용종료일 설정<br/>변경 전·후 단가 이력 자동 기록]
    Q2 -->|없음 — 최초 등록| Q4[신규 단가 이력 등록]
    Q3 --> Q5[신규 단가를 유효 단가로 설정]
    Q4 --> Q5

    Q5 --> Q6[견적 작성 시 거래처별 유효 단가 자동 적용<br/>과거 견적은 당시 단가 스냅샷 유지]
    Q6 --> E([종료])
```

> **핵심:**
> - 하나의 자재는 복수의 거래처에서 구매할 수 있다 (N:M 관계).
> - 거래처·단가 정보는 자재 마스터에 포함하지 않고, 자재-거래처 매핑에서 별도 관리한다.
> - 단가는 구매/발주 시점에 결정되며, 자재 마스터 저장과는 무관하다.
> - 견적서는 작성 시점의 유효 단가를 스냅샷으로 저장하여, 이후 단가 변경이 기존 견적에 소급 적용되지 않는다 (FR-CM-005-06 해결).

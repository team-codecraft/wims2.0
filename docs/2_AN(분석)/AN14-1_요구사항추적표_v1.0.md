## 1. 개요

### 1.1 목적

본 문서는 WIMS 2.0 프로젝트의 모든 요구사항(122건)에 대해 분석 → 설계 → 구현 → 테스트 단계별 산출물 간 추적성(Traceability)을 확보하기 위한 요구사항 추적표(RTM, Requirements Traceability Matrix)이다.

### 1.2 적용 범위

- **대상:** 기능 요구사항 76건 + 비기능 요구사항 46건 = 총 122건
- **추적 방향:** 양방향 추적 (Forward: 요구사항→설계→구현→테스트 / Backward: 테스트→구현→설계→요구사항)
- **갱신 주기:** 스프린트 종료 시점마다 갱신 (S1~S15)

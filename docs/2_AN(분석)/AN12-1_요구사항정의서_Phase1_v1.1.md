## 1. 개요

### 1.1 목적·범위

Phase 1 (제품관리 PM + 공통 CM) 기능·비기능 요구사항 SOT. v1.1 에서 용어사전 v1.3/v1.4·DE11-1 v1.2·DE22-1 v1.5→v1.6·DE24-1 v1.8→v1.9·DE35-1 v1.5 반영. v1.1-r2 (2026-04-23) 에서 DE22-1 의 sections/ 분산 구조 패턴 적용 + 부록 A v1.0 계승 FR 을 PM/CM 본문에 통합하여 서브시스템별 완전한 요구사항 목록 확보.

Phase 2 (ES/OM/MF/FS) 요구사항은 [[AN12-1_요구사항정의서_Phase2_v1.0]] 별도 관리.

### 1.2 분산 구조

메인 인덱스(본 파일) + 섹션 파일 3개 (`sections/00_공통_원칙_ID체계`, `PM_제품관리`, `CM_공통`). 각 섹션은 독립 편집 가능하며 Obsidian wikilink·그래프뷰로 연결.

## 4. v1.1 개정 핵심 포인트

### 4.1 표준 BOM 삼위일체 정립

EBOM(자재구성) + MBOM(공정구성) + Config(옵션구성) 를 **불가분 묶음**으로 재정의하고 단일 `standardBomVersion` 축으로 버전 관리. 각 UI 용어 확정: 자재구성·공정구성·옵션구성·옵션별규칙·확정구성표.

### 4.2 4계층 제품 분류 고정

`productClassPath = L1 대분류 / L2 계약구분 / L3 유리사양 / L4 치수크기`. 자유 3단계 트리 폐기, 4축 체크박스 필터 및 modelCode 세그먼트 디코딩(FR-PM-018).

### 4.3 RuleEngine + AST 캐시 + frozen 불변성

`UNIQUE_V1` 산식 평가기(FR-PM-024), AST 캐시 기동 60초(NFR-MT-PM-001), 5,000 rule 기준 p95 ≤ 100ms(NFR-PF-PM-003), frozen 후 100% 불변(NFR-RL-PM-001).

### 4.4 BOM_RULE 3뷰 UX (템플릿·결정표·시뮬)

초보용 템플릿 갤러리(FR-PM-025) / 조감용 결정표(FR-PM-026) / 저장 전 검증 시뮬레이터(FR-PM-027) 3 독립 화면 체계. DE22-1 v1.6 에서 SCR-PM-021/022/023 독립 URL·SCR ID 부여.

### 4.5 MES 응답 DTO 8필드 + supplyDivision 쿼리

DE24-1 v1.8 반영(FR-PM-013): `cutDirection`, `cutLength`, `cutLength2`, `cutQty`, `actualCutLength`, `supplyDivision`, `frozen`, `itemCategory`. `/cutting-bom/**` 엔드포인트 철회 확정.

### 4.6 파생제품·공급망 정식화

파생제품(`derivativeOf`, `derivativeKind` 4종) Variant BOM 자동 상속(FR-PM-019). 다이스북 개정판·DIES_SUPPLIER 역할·ITEM_SUPPLIER 다대다(FR-PM-023).

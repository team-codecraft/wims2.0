# 공통 원칙·레이아웃

> [!abstract]
> 디자인 시스템·인터랙션 규칙·반응형 브레이크포인트·GNB/LNB/공통 컴포넌트 등 Phase 1 전 화면이 공유하는 기반을 정의한다. 개별 화면 명세는 다른 섹션 파일에 분산.

## 3. 공통 레이아웃

> **설계 결정 (FR-CM-003):** 다중 창은 전용 화면이 아닌, 메인 콘텐츠 + 우측 상세 패널(리사이즈) 구조로 전역 구현. 모든 목록→상세 화면에서 동일 적용.

### 3.1 전체 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│ [Header] GNB — 로고, 메뉴, ⌘K 검색, 알림벨, 사용자 메뉴   │
├────────┬─────────────────────────┬───────────────────────┤
│ [LNB]  │ [Main Content]          │ [Detail Panel]        │
│ 56px   │ flex: 1                 │ 기본 400px, 320~600   │
├────────┴─────────────────────────┴───────────────────────┤
│ [Footer] 버전·저작권                                       │
└──────────────────────────────────────────────────────────┘
```

### 3.2 GNB

| 영역 | 내용 |
|------|------|
| 좌측 | WIMS 로고 + 시스템명 |
| 중앙 | 프로젝트 · 자재관리 · 제품관리 · **공정관리** · 거래처관리 · 시스템관리 (Phase 1, 6개 메뉴) |
| 우측 | **⌘K 글로벌 검색** · 알림벨(NotificationBell) · 사용자 아바타+이름 · 로그아웃 |

> **v1.6 변경:** 공정관리를 별도 GNB 로 분리 (기존 5개 → 6개). 우측에 Command Palette(⌘K) 글로벌 검색 상시 노출.
> **Phase 2 추가:** 견적설계·발주관리·제조관리 (GNB 또는 프로젝트 상세 내 탭).

### 3.3 LNB 상세 스펙

GNB 진입 시 좌측 서브메뉴를 아이콘+텍스트로 표시. **접힘 56px / 펼침 200px.** 각 GNB 별 LNB 트리 구조는 아래와 같다.

| GNB | LNB 서브메뉴 | 대응 화면 |
|-----|------------|----------|
| 프로젝트 | 프로젝트 목록 / 내 프로젝트 / 관심 / 종료 | SCR-PM-015 (탭) |
| 자재관리 | 자재 목록 / 자재 등록 | SCR-PM-001/002 |
| 제품관리 | 제품 목록 / 제품 등록 / 파생제품 / BOM 트리뷰 / 옵션구성 / 옵션별규칙 (템플릿 갤러리·결정표·시뮬레이터·버전 관리) | SCR-PM-010~023 |
| 공정관리 | 공정 관리 | SCR-PM-007 |
| 거래처관리 | 거래처 관리 | SCR-PM-004 |
| 시스템관리 | 사용자 관리 / 그룹 관리 / 코드 관리 / 시스템 설정 | SCR-CM-003/005/006/007 |

> [!info]
> BOM·옵션별규칙은 제품 상세 진입이 주 경로이므로 LNB 는 **바로가기 수준**으로 표시. 실제 편집은 제품 상세의 각 탭(자재구성·공정구성·옵션구성·옵션별규칙)에서 수행한다.

### 3.6 글로벌 검색 범위

⌘K Command Palette 가 검색하는 대상을 아래 표에 명시한다.

| 대상 | 필드 | 점프 목적지 |
|------|------|-----------|
| 제품 | productCode, productName, modelCode | SCR-PM-012 |
| 자재 | itemCode, itemName | SCR-PM-003 |
| 프로젝트 | projectNo, projectName, siteAddress | SCR-PM-016 |
| 거래처 | partnerName, bizRegNo | SCR-PM-004 (상세패널 자동 오픈) |
| 공정 | processCode, processName | SCR-PM-007 |
| 파생제품 | derivativeCode, derivativeName | SCR-PM-017 |

**동작 규칙**

- 키워드 타이핑 시 **디바운스 300ms** 후 서버 검색
- 카테고리별 **그룹화 표시** (최대 5건/카테고리)
- 키보드: ↑↓ 이동, Enter 선택, Esc 닫기

**권한 기반 결과 필터링 (DEC-S12, 보안)**

- 서버는 현재 사용자 ROLE scope 내 대상만 반환 (정본: [[DE11-1_소프트웨어_아키텍처_설계서_v1.2|DE11-1]] §7.3)
- `ROLE_MES_READER` → 본인 조회 권한 범위의 제품만
- `ROLE_PM_VIEWER` → 모든 PM 대상 조회 가능 / 편집 X
- `ROLE_PM_EDITOR` · `ROLE_PM_ADMIN` → 전체 PM 범위
- `ROLE_RULE_EDITOR` → 규칙 대상(제품·옵션구성) 범위
- `ROLE_ESTIMATE_EDITOR` → Phase 2 (ES scope)
- `ROLE_ADMIN` → 전체

**딥링크 복원 규격**

- 점프 시 URL 에 `?from=search&q={query}&hl={highlightId}` 파라미터 전달
- 대상 화면은 해당 `hl` 노드/행을 **하이라이트 + 자동 스크롤**
- **뒤로가기**: 검색 결과 패널 상태 복원 (쿼리·선택 범주·스크롤 위치)
- **최근 검색어**: `localStorage` 10건 저장 (사용자별 key — `wims.search.recent.{userId}`)

### 3.8 SCR 상태 4종 표준 (loading · empty · error · success)

> **DEC-S2.** 모든 SCR 섹션은 본 4상태 행을 **필수 포함**한다. 상태별 UI 기본 규격은 아래와 같다.

| 상태 | 기본 규격 | 상세 |
|------|----------|------|
| **loading** | 스켈레톤 UI 기본 | DataTable row 스켈레톤 · 상세패널 blur. 네트워크 > 500ms 시 표시. **타임아웃 10초 시 error 전환** |
| **empty** | §3.4 `EmptyState` 컴포넌트 재사용 | 문구 **3요소** = 원인 + 유도 CTA + 도움말 링크. 예: "등록된 자재가 없습니다. [+ 자재 등록] 혹은 [Excel 가져오기]." |
| **error** | 상태 코드별 분기 | 4xx → **인라인 배너** + [재시도][닫기] / 5xx → **전면 error page** + [새로고침][홈]. RFC 7807 `title`·`detail` 은 사용자 노출, `instance` (trace id) 는 콘솔·[문의] 버튼 첨부 |
| **success** | 토스트 3초 | 실행취소 있는 경우 5초. 목록 변경 시 해당 행 **하이라이트 2초** (pulse) |

> [!note]
> 개별 SCR 명세 표에 `| loading | empty | error | success |` 4행을 필수 기재. 본 공통 규격을 따르지 않는 경우에만 개별 섹션에서 오버라이드 명시.

### 3.9 공통 분기 처리 (409 · 429 · 권한 · 세션)

| 분기 | UX 처리 | 상세 |
|------|--------|------|
| **409 Conflict (optimistic locking)** | 경고 토스트 "다른 사용자가 먼저 저장했습니다" + **merge helper 모달** | 3-way diff (내 변경 / 상대 변경 / 병합) · 3 선택지 [유지(내 것)][덮어쓰기(상대 것)][병합] · `If-Match`/`ETag` 헤더 활용 |
| **429 Too Many Requests** | 토스트 "잠시 후 다시 시도" | `Retry-After` 헤더 기반 **카운트다운 표시** · 클라이언트 쪽 debounce 강화 (동일 요청 300ms 합치기) |
| **401 Unauthorized** | 로그인 만료 모달 → 재로그인 | 재로그인 성공 시 **원래 화면·작업 유지**. 자동저장(FR-CM-005) 내용 복원 |
| **403 Forbidden** | 전면 "접근 권한 없음" 페이지 | 요청된 권한 기재 (예: `ROLE_PM_EDITOR` 필요) · [홈으로][관리자 문의] CTA |
| **세션 만료 15분 전** (FR-CM-001) | 모달 [연장][로그아웃] | 자동저장 중이면 "작업 중인 내용이 자동 저장되었습니다" 안내 · 만료 후 재로그인 시 **FR-CM-005-09 자동 복구** |

### 3.10 결번 SCR 레거시 URL 처리

> **DEC-S5.** v1.5 → v1.6 에서 통합·폐기된 SCR 의 구 URL 을 301 Permanent Redirect 로 보존한다.

**결번 4건 및 redirect 매핑 (FE 라우터 레벨)**

| 구 SCR | 신 SCR | 구 URL → 신 URL |
|--------|-------|----------------|
| SCR-PM-005 | SCR-PM-004 | `/partners/:id/detail` → `/partners/:id` |
| SCR-PM-008 | SCR-PM-007 | `/processes/new` → `/processes/:code#basic` |
| SCR-PM-009 | SCR-PM-007 | `/processes/:code/pricing` → `/processes/:code#pricing` |
| SCR-CM-004 | SCR-CM-003 | `/admin/users/:id/groups` → `/admin/users/:id#groups` |

**운영 규칙**

- **301 Permanent Redirect** 사용 (검색엔진·북마크 보존 신호)
- 외부 북마크·문서 참조 보존
- Analytics 로 redirect 빈도 **모니터링** → **1년 후 폐기 평가**

### 3.11 삭제 전 의존성 검사 · 더블체크

> **DEC-S11.** 모든 마스터 삭제 API 는 **3단계 게이트**를 거친다.

**3단계 게이트**

1. `GET /{resource}/:id/dependencies` — 연결 대상 카운트 (예: BOM 5건·견적 12건)
2. **영향도 확인 모달** — 차단 대상 (`RESTRICT`) vs 캐스케이드 (`CASCADE`) 분리 표시
3. **사용자 이름 타이핑 확인** (3건 이상 연결 시) + [삭제] 버튼 활성화

**Soft delete 기본 정책**

- `deleted_at` 컬럼 사용 · **30일 복원 가능** · 감사 로그(audit log) 필수
- 복원 UI: 각 목록에 **"휴지통" 필터 탭** 제공
- 30일 후 **hard delete** (cron 배치)

**409 응답 메시지 예시**

> "이 자재는 N개 BOM 에서 사용 중입니다. 먼저 연결 해제 후 삭제하세요." + **연결된 BOM 목록 링크**

## 4. 접근성·국제화 원칙

### 4.1 a11y 원칙 (WCAG 2.1 AA)

- [ ] 모든 인터랙티브 요소 **키보드 접근 가능** (Tab 순서, Enter/Space 활성화)
- [ ] **포커스 시각적 표시** (2px outline, accent 컬러)
- [ ] 스크린리더용 **ARIA label 필수** (SR-only 텍스트로 아이콘 버튼·상태 안내)
- [ ] **색상 단독 정보 전달 금지** (아이콘·텍스트 병행 — 예: 오류를 빨간색만으로 표시하지 않음)
- [ ] **명도 대비 4.5:1** (본문) / **3:1** (대형 텍스트, 18px 이상 또는 14px bold)
- [ ] **자동 이동 콘텐츠 일시정지 가능** (캐러셀·자동 갱신 대시보드 등)

### 4.2 a11y 필수 체크리스트 (WCAG 2.1 AA)

> **DEC-S4.2.** 각 SCR 섹션은 본 체크리스트 **준수 명시**를 필수로 기재한다 (예: "a11y: §4.2 준수").

- [ ] **키보드 순서**: Tab / Shift+Tab 논리적 진행, **Skip to main (Alt+S)** 제공
- [ ] **포커스 트랩**: 모달·드로어 진입 시 내부 포커스 유지, **ESC 닫기** 지원
- [ ] **ARIA**: `role` · `aria-label` · `aria-live` (비동기 결과 공지) 필수
- [ ] **색상 대비**: 본문 **4.5:1**, 큰 글자(18pt+ / 14pt+ bold) **3:1**, **색 단독 정보 금지** (아이콘·텍스트 병기)
- [ ] **에러 메시지**: `aria-describedby` 로 input 연결 → screen reader **자동 낭독**
- [ ] **동작 완료 알림**: `aria-live="polite"` region (예: "저장되었습니다")

### 4.3 국제화 (i18n)

- Phase 1 은 **한국어 단일**. 다국어 대응은 Phase 2 로 이연 (참조: [[WIMS_용어사전_BOM_v1.4]] §개방이슈)
- UI 문자열은 **i18n key 로 추출 권장** (하드코딩 지양, 추후 locale 파일 전환 대비)
- 숫자·일자·통화 포맷은 `Intl` API 경유로 통일

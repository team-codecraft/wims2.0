# DE34-1 표준 데이터사전 v1.0

**문서코드:** DE34-1
**버전:** v1.0
**작성일:** 2026-04-27 (S2 8일차)
**작성자:** 김지광 (PM, 코드크래프트)
**상태:** draft (Gate 2 진입 전 초판)

> [!abstract] 문서 개요
> 본 문서는 PP12-1 v2.2 §1 ●필수로 채택된 **"표준 데이터사전 — 자재 코드 체계 표준화 포함"** 산출물이다. WIMS 2.0 BOM 도메인 데이터의 표준 식별 체계를 한 곳에서 조망하기 위한 **슬림 인덱스**이며, 본 문서가 단독 SOT로 책임지는 영역은 **§2 자재 코드 체계 표준화** 한 절뿐이다. 컬럼 사전·코드 카탈로그는 각각 [[DE33-1_DB물리스키마_설계서_v1.3]]·[[WIMS_용어사전_BOM_v1.4]] 가 SOT 이며, 본 문서는 인덱스·인용만 제공하여 SOT 분열을 차단한다.
>
> **범위:** Phase 1 PM(제품관리) 서브시스템 깊이까지. Phase 2 (ES·OM·MF·FS) 도메인 코드 체계는 §2.6·§5에 [TBD] 자리만 확보.

---

## §1 개요

### 1.1 목적

본 문서는 다음 세 가지 목적을 갖는다.

1. **PP12-1 v2.2 §1 ●필수 산출물 충족** — 설계 단계 표준 데이터사전. 콘텐츠 명시: "자재 코드 체계 표준화 포함".
2. **자재 코드 체계 단독 SOT 제공** — 원자재·부자재·반제품·위치 인스턴스 코드 부여 규칙을 한 문서로 모은다. 시드 입력(DE41 §3.4) · ITEM 마스터 등록(SCR-PM-001~003) · BOM 평가(RESOLVED_BOM)에서 동일 규칙 참조.
3. **컬럼 사전·코드 사전 통합 인덱스** — DE33-1 23 엔티티 컬럼 COMMENT 117건과 용어사전 §4-혹은-§13 enum 카탈로그를 도메인별 §링크로 연결하여 BE·FE·DBA·BA의 진입점을 단일화한다.

### 1.2 범위

| 영역 | Phase 1 (현재) | Phase 2 (후속) |
|---|---|---|
| 자재 코드 체계 | ✅ §2 단독 SOT — 87종 자재·11개 협력업체·31공정·10위치 | [TBD] §2.6 — 견적·발주·제조·실측 도메인 추가 코드 |
| 컬럼 사전 인덱스 | ✅ §3 — DE33-1 v1.2 23 엔티티 인덱스 | [TBD] — Phase 2 신설 엔티티 등록 시 추가 |
| 코드 도메인 사전 인덱스 | ✅ §4 — 용어사전 v1.4 enum·금지어 인덱스 | [TBD] — Phase 2 OPTION_GROUP 카테고리 추가 |
| Phase 2 도메인 코드 | — | [TBD] §5 자리확보 |

### 1.3 SOT 분담

본 문서가 인덱스 위치에 그치는 두 영역과, 단독으로 책임지는 한 영역을 명확히 구분한다. SOT 분담을 어기는 본문 복사는 금지한다.

| 영역 | SOT 문서 | 본 DE34-1 위치 |
|---|---|---|
| 컬럼 사전 (테이블·컬럼·타입·COMMENT) | [[DE33-1_DB물리스키마_설계서_v1.3]] §3 (23 엔티티 DDL) | §3 인덱스만 (인용 wikilink·매트릭스) |
| 코드 도메인 사전 (enum·금지어·표준명) | [[WIMS_용어사전_BOM_v1.4]] §1·§2·§3·§13 | §4 인덱스만 (자주 쓰는 enum 한눈 표) |
| **자재 코드 체계 표준화** | **본 문서 §2** | **§2 단독 SOT** |
| BOM Level 5단 정의 | [[WIMS_용어사전_BOM_v1.4]] §2.1 | §2.5에서 매핑 인용만 |
| 위치구분 카탈로그 10종 | [[WIMS_용어사전_BOM_v1.4]] §5.1 | §2.1 위치 인스턴스에서 인용 |

> [!warning] SOT 분열 차단
> §3·§4의 본문 복사를 금지한다. 컬럼명·enum 값을 본 문서에 베껴 두면 SOT가 두 곳이 되어 불일치가 발생한다. 변경 시 SOT 문서 한 곳만 갱신하고, 본 문서는 인덱스 wikilink 만 유지한다.

## §2 자재 코드 체계 표준화 (본 문서 단독 SOT)

> [!info] §2의 위상
> 본 절은 PP12-1 v2.2 §1 명시 콘텐츠 "자재 코드 체계 표준화" 의 **단독 SOT** 다. 자재 마스터 87종 (DE35-1 §4) · 시드 입력 (DE41 §3.4) · BOM 평가 (RESOLVED_BOM) · 화면 (SCR-PM-001~003) 모두 본 절 규칙을 참조한다. 신규 자재 식별 시 본 절 갱신을 선행하고, 그 후 DE35-1 §4 카탈로그·DE41 §3.4 시드를 갱신한다.

### 2.1 자재 코드 부여 규칙

WIMS 2.0 자재 코드는 **5계열** 규칙을 따른다. 모든 코드는 ASCII 영숫자·하이픈만 허용 (대소문자 구분 없음, 저장 시 대문자 정규화).

#### 2.1.1 원자재 (Raw Material, L5 ROH) — `UNI-A{치수}-{유형}`

| 패턴 | 예 | 의미 |
|---|---|---|
| `UNI-A{치수}-{유형}` | `UNI-A225-101` | 225mm 1등급 미서기 후렘 |
| `UNI-A{치수}-{유형}{변형}` | `UNI-A225-101A` | 225 1등급 변형 A |
| `UNI-P{치수}-{유형}` | `UNI-P225-401` | 225 PVC 단열재 401형 |
| `YJN-SL{치수}A-{유형}` | `YJN-SL160A-101` | 160 마스 — YJN 계열 |
| `DH-XXXX-X` | `DH-5555-1` | 대화 계열 (외주 압출 일부) |
| `SC-XXXX` / `ZZ-ZZ##` | `SC-1911`, `ZZ-ZZ03` | 특수 압출재 시리얼 |

**치수**: 225 / 160 / 140 (Phase 1 미서기 이중창 3 라인). **유형**: 101 / 401 / 901 / 10H / 90H 등.

#### 2.1.2 부자재 (Component, L4 Part) — `NN-XXXX` (4자리 시리얼)

| 접두 NN | 계열 | 예 | 비고 |
|---|---|---|---|
| `01-` | 나사·피스 (대아볼텍) | `01-0059` | 4자리 시리얼 |
| `02-` | 가람정밀 계열 (가이드·브라켓·연결재·이탈방지·후렘이탈방지 등) | `02-0077`, `02-0091`, `02-0097` | 가장 다종 (가람 16건 + α) |
| `04-` | 모헤어·가스켓·롤러 | `04-0002` | 삼오섬유·기타 |
| `X-001` | 외주 통합 코드 | `X-001` | 단일 자재 (Q14 X-/SUB- 이중체계 정리, X- 통일) |

> [!note] X-/SUB- 정리
> 2026-04-24 보완본에서 `X-001` 만 유지하고 `SUB-*` 계열 전면 폐기. (질의 Q14 해소)

#### 2.1.3 반제품 (Semi-finished, L2 HALB / L3 Sub-Asm) — 4계열

| 패턴 | 의미 | 예 |
|---|---|---|
| `F-XXXX` | 완제품 / Frame 완제품 | `F-0001` |
| `HC-XXX` | 가공 반제품 (Hi-Cut, 절단·홀가공 후) | `HC-001`, `HC-002` |
| `HX-XXXX` | 조립 반제품 (eXecuted assembly) | `HX-0001` |
| `{원자재코드}-H` | 널링 반제품 (널링 공정 후) | `UNI-A225-10H`, `UNI-A225-90H` |

> [!note] Phantom 아님 — 실체 반제품
> 위 4계열은 모두 자재코드 부여 대상이며 `is_phantom=FALSE`. 재고 보유·이동 추적·비용 산정 모두 적용된다. (질의 Q9 해소)

#### 2.1.4 위치 인스턴스 (Location Code, 10종)

[[WIMS_용어사전_BOM_v1.4]] §5.1 SOT 인용. 본 절은 자재코드 체계 관점의 매핑 규칙만 정의.

```
H 계열 (수평) 7종 : H01, H02, H02-1, H02-2, H03, H03-1, H03-2
W 계열 (수직) 3종 : W01, W02, W03
```

**Sub-position 접미사 규칙**: 동일 위치 내 분할(`-1`, `-2`)은 부모 위치(접미사 없는 코드)와 자재코드는 동일하되 `BOM_ITEM_LOCATION.location_code` 컬럼으로만 분리. 즉 `UNI-A225-101 @ H03` 와 `UNI-A225-101 @ H03-1` 은 동일 ITEM·다른 위치 인스턴스.

**품목별 사용 패턴**:
- 후렘: 10종 전체 사용
- 문짝: H01·H02·H03 + W01·W02·W03 (6종)
- 방충망: H01·W01 (2종)

#### 2.1.5 ITEM 카테고리 (`itemCategory` enum)

용어사전 §1 인용. 자재 코드 체계와 직교 차원이며, 동일 자재가 어느 카테고리에 속하는지를 표시한다.

```
PROFILE | GLASS | HARDWARE | CONSUMABLE | SEALANT | SCREEN
```

자재 코드 체계 관점 매핑:
- 원자재 `UNI-A*` → 대부분 `PROFILE`
- 원자재 `UNI-P*` → `CONSUMABLE` (단열재)
- 부자재 `01-*`, `02-*` → `HARDWARE` 또는 `CONSUMABLE`
- 부자재 `04-*` → `SEALANT` 또는 `SCREEN`
- 반제품(F/HC/HX/-H) → 카테고리 미정의 (조립체)

### 2.2 자재 마스터 87종 인덱스

상세 카탈로그 SOT는 [[DE35-1_미서기이중창_표준BOM구조_정의서_v1.6]] §4 (제품 카탈로그 25종 + Level 전개표). 본 절은 코드 체계 관점의 분포 요약만 제공.

| 구분 | 수량 | Level | 패턴 분포 |
|---|---|---|---|
| 원자재 | 40 | L5 ROH | `UNI-A*` 다수 + `UNI-P*` + `YJN-*` + `DH-*` + `SC-*` + `ZZ-*` |
| 부자재 | 47 | L4 Part | `01-*` (소수) + `02-*` (가람 16+α) + `04-*` + `X-001` |
| 반제품 | (별도 집계) | L2 HALB / L3 Sub-Asm | `F-*` + `HC-*` + `HX-*` + `*-H` |
| 합계 | **87 (원+부)** | — | 코드 미부여 0건 (Q2 해소) |

> [!info] 기준
> 87종 = 원자재 40 + 부자재 47. 반제품은 별도 집계 (제품·BOM 라인업에 따라 가변).
>
> 단가 보유: 42 / 87 (48%). 부자재는 대부분 단가 보유, 원자재(압출재)는 내부 생산이라 단가 미보유. 시드 입력 시 단가 NULL 허용.

자세한 자재별 코드·치수·단가는 [[7_유니크시스템BOM_26.04.24_분석]] §4 Gain D + 보완본 `품목별 원부자재 사용량` 시트 (413행) 참조.

### 2.3 협력업체 코드 — 11개사

`ITEM_SUPPLIER` 매핑 (DE33-1 §3.19 SOT) 의 시드 대상 협력업체 11개사. 코드 부여는 시드 입력 단계 (DE41 §3.4) 에서 결정한다.

|---|---|---|---|
| 1 | 가람정밀 | 16 | 가이드·브라켓·연결재·이탈방지 (`02-*` 다수) |
| 2 | 에이스이노텍 | 8 | 크리센트·브라켓 |
| 3 | 대아볼텍 | 5 | 나사·피스 (`01-*`) |
| 4 | 제스트 | 4 | — |
| 5 | 전진기업 | 3 | 풍지판 |
| 6 | 성문에스피 | 3 | 가스켓 |
| 7 | 중앙자석 | 1~2 | 자석류 |
| 8 | 유니크홀딩스 | 1~2 | 그룹 내부 공급 |
| 9 | 삼오섬유 | 1~2 | 모헤어 |
| 10 | KS볼트 | 1~2 | 특수 볼트 |
| 11 | 삼일다이캐스팅 | 1~2 | 다이캐스팅 부품 |

> [!note] 단가 이중 컬럼 미반영
> 보완본 `단가(U)` · `단가(H)` 이중 컬럼은 유니크시스템 그룹 두 회사 단가 병기다. WIMS 2.0 은 단일 `unitPrice` 체계이므로 시드 입력 시 한쪽만 채택한다. (질의 Q-추가, [[7_유니크시스템BOM_26.04.24_분석]] §1 abstract 주의 사항)

### 2.4 공정 코드 카탈로그

용어사전 §1 `processCode = PRC-{유형}-{순번}` 규칙 적용. 품목별 표준 공정은 [[DE35-1_미서기이중창_표준BOM구조_정의서_v1.6]] §7A SOT 인용. 본 절은 코드 체계 관점에서 공정 종류·총수만 요약.

| 품목 | 공정 수 | 순번 범위 | 비고 |
|---|---|---|---|
| 후렘 (Frame) | **13공정** | 0 (널링) ~ 12 (포장) | `PRC-FRM-00` ~ `PRC-FRM-12`. 0번 널링은 사전공정 |
| 문짝 (Sash Door) | **12공정** | 0 (널링) ~ 11 (포장) | `PRC-SASH-00` ~ `PRC-SASH-11` |
| 방충망 (Screen) | **6공정** | 1 (절단) ~ 6 (포장) | `PRC-SCR-01` ~ `PRC-SCR-06`. 널링 없음 |

**공정 종류 catalog**: 널링 · 절단 · 피스홀가공 · 배수홀가공 · 풍지판부착 · 이탈방지장착 · 가네고부착 · 나사체결 · 양후렘연결재결합 · 탄성레일바절단 · 레버부가공 · 모헤어삽입 · 크리센트부착 · 코너브라켓결합 · 고리캡절단 · 고리캡가공 · 가이드부착 · 크리센트고리부착 · 가네고결합 · 브라켓결합 · 조립 · 포장. (총 22종 — 일부 품목 공통)

> [!note] CT (표준 작업시간)
> 유니크시스템 현업은 공정별 표준 작업시간을 기존에 관리하지 않음 (2026-04-24 PO 확인, 질의 Q19-b N/A). WIMS 설계에서도 필수 필드 아님. Phase 2 MES 연동 시 실적(observed CT) 역산 검토.

### 2.5 BOM Level 5단 (FERT/HALB/Sub-Asm/Part/ROH) — 코드 체계 매핑

용어사전 [[WIMS_용어사전_BOM_v1.4]] §2.1 Level 정의 SOT 인용. 본 절은 자재코드 체계 관점에서 각 Level이 어느 코드 패턴에 매핑되는지만 제시.

| Level | 명칭 | SAP 대응 | 본 §2 코드 패턴 |
|---|---|---|---|
| L1 | 완제품 | FERT | `PRD-*` (modelCode = `DHS-AE225-D-1` 등) |
| L2 | 반제품 / 모듈 | HALB | `F-*` 완제품 / `ASY-*` 조립체 (DE35-1 §4) |
| L3 | 서브조립 | — | `HC-*` 가공 / `HX-*` 조립 / `-H` 널링 |
| L4 | 부품 | — | `01-*` / `02-*` / `04-*` / `X-001` 부자재 47종 |

대표 금지·대체:
- `productVersion`/`configVersion` → `standardBomVersion` (단일 버전축)
- `cuttingBomId`/`CUTTING_BOM` → MBOM + 절단 속성 (v1.1 철회)

### 4.3 식별자 패턴 인덱스

[[WIMS_용어사전_BOM_v1.4#1. 식별자]] §1 SOT. 본 항은 식별자 명·DB·도메인·API 4계층 매핑이 필요할 때 진입점.

핵심 식별자:
- `standardBomId` / `standardBomVersion` — 표준BOM 묶음 단일 버전
- `resolvedBomId` = `RBOM-{standardBomId}-sbv{N}-{optionsHash}` (결정적 생성)
- `itemCode` — 본 §2 자재 코드 체계 단독 SOT 적용 대상
- `processCode` — `PRC-{유형}-{순번}` (본 §2.4 매핑)
- `locationCode` — H01~W03 (본 §2.1.4 매핑)
- `modelCode` — 제품 PK (예: `DHS-AE225-D-1`)

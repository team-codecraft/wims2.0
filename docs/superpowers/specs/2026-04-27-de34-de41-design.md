# DE34-1 + DE41 산출물 작성 설계안

## 1. 배경

PP12-1 v2.2 §1 ●필수로 채택된 DE 단계 산출물 중 본 시점(2026-04-27, S2 8일차) 미작성 2건을 brainstorming으로 설계 확정한다.

| 산출물 | PP12-1 §1 적용 | 시점 | 상태 |
|---|---|---|---|
| DE34-1 표준 데이터사전 | ●필수 | S2 (M2 1주차) | 미작성 |
| DE41 데이터 전환 계획서 | ●필수 (콘텐츠 재정의: 초기 데이터 시드 입력 정책서) | S2 (M2 2주차) | 미작성 |

**Gate 2 진입 조건** (PP12-1 v2.2 §4 L122) — "BOM 논리/물리 ERD 확정 / MES API 규격서 확정·MES 팀 서명 / 화면 설계서 확정 / **데이터 전환 매핑 정의 완료**" — DE41 작성으로 충족.

**전제 정책** (PP12-1 v2.2·PP00 v1.3·PP-SCH v1.1·AN22-1 v1.1 동시 반영, 2026-04-27):
- WIMS 2.0은 **빈 DB로 시작**, 기존 As-Is 데이터 마이그레이션 미실시
- 운영 시점에 **마스터 시드 입력**: 자재 87종·제품 25종·사용자 14명·코드 카탈로그·협력업체 11개사
- AN22-1 v1.1 §5는 "참고용 As-Is 분석" 성격으로 재기술 완료

## 2. 합의된 brainstorming 결정 (Q1~Q4)

|---|---|---|---|
| Q1 | 산출물 깊이 | **C 중간** | Phase 1 PM 깊이 / Phase 2 [TBD] 마커 |

§1 개요

       - 원자재: UNI-A{치수}-{유형} 패턴

  2.3 협력업체 코드 11개사
  2.4 공정 코드 카탈로그 (후렘13·문짝12·방충망6)
  2.5 BOM Level 5단 (FERT/HALB/Sub-Asm/Part/ROH) — 정의는 용어사전 인용
  2.6 Phase 2 도메인 코드 체계 [TBD]

§3 컬럼 사전 인덱스

§4 코드 도메인 사전 인덱스

§5 Phase 2 도메인 [TBD]
  - 견적·발주·제조·현장실측 코드 체계

## 4. DE41 초기 데이터 시드 입력 정책서 v1.0 — 구조

**파일**: `docs/3_DE(설계)/DE41_초기데이터시드입력정책서_v1.0.md`
**분량 추정**: 500~600행
**산출물 코드**: DE41 (PP12-1 v2.2 §3 묶음 표기 부합. DE41-1·DE41-2를 §2·§3로 흡수)

```
frontmatter

§1 개요
  1.1 목적 — PP12-1 v2.2 §1 ●필수 / Gate 2 진입조건 / 마이그 미실시·빈DB·시드 입력 정책
  1.2 범위 — 빈 DB 시작 시 입력해야 할 마스터 시드 데이터 정의
  1.3 As-Is 참고 분류 — AN22-1 v1.1 §5.2 7개 도메인 요약 (참고만, 마이그 미실시)
  1.4 참조 — DE33-1 v1.2 / DE34-1 v1.0 / AN22-1 v1.1 / BOM_26.04.24 보완본

§2 시드 입력 계획 (DE41-1 영역)
  2.1 입력 전략 — Phase 1 일괄 시드 (S5 D-Day) / Phase 2 점진 입력
  2.2 입력 일정 — S2 시드 정의 → S3~4 시드 데이터 준비 → S5 D-Day
  2.3 입력 도구 — Flyway 시드 (코드 카탈로그) + 관리자 화면 (사용자) + Excel→CSV 일괄 (자재·제품)
  2.4 검증 정책 — 전수 검증 (자재 87종·제품 25종·사용자 14명·코드값 전체)
  2.5 권한 / 책임 — DBA·BE·BA·PM 분담
  2.6 리스크 — 시드 데이터 품질·운영 시작 후 추가 입력

★ §3 도메인별 시드 입력 명세 (DE41-2 영역)
  3.1 시드 매트릭스 요약 (도메인 × 엔티티 × 시드 규모 × 입력 방식)
  3.2 인증·사용자 시드 (Phase 1)
       - 사용자 14명 (TFT 명단) + JWT 패스워드 신규
       - 7 표준 ROLE (DE11-1 §7.3) + 권한 매핑
  3.3 프로젝트 시드 (Phase 1)
       - 운영 시점 신규 등록 (As-Is 미이관, 시드 0건)
  ★ 3.4 자재·제품 시드 (Phase 1, 최고 깊이)
       - 자재 87종 (BOM_26.04.24 보완본 → CSV)
       - 제품 25종 (후렘 18·문짝 6·방충망 1, modelCode 카탈로그)
       - 협력업체 11개사
       - 다이스북·BOM 트리 시드
       - 자재코드 체계 정합성 (DE34-1 §2 참조)
  3.5 시스템·공통 시드 (Phase 1)
       - 코드 카탈로그 (단위·분류·공정·위치 등)
       - 시스템 설정 12항목 기본값
  3.6 견적설계 시드 [TBD — Phase 2]
  3.7 계약·발주 시드 [TBD — Phase 2]
  3.8 도면·CAD 시드 [TBD — Phase 2]

§4 시드 입력 표준 규칙
  4.1 자재코드 표준 패턴 적용 (DE34-1 §2 인용)
  4.2 ROLE 매핑 (DE11-1 §7.3 7표준)
  4.3 인코딩·문자셋 (UTF-8)
  4.4 NULL·기본값 정책

§5 (없음) — 신규 입력이므로 데이터 정제 불필요

§6 검증 정책
  6.1 시드 입력 검증 SQL 카탈로그 (Phase 1 깊이)
  6.2 검증 결과 보고서 템플릿 (IM 단계 "초기 데이터 시드 입력 결과서" 입력)
  6.3 합격 기준 (시드 row count 100% / FK 무결성 100% / 표본 정성검증)

§7 미해결 이슈 / 후속 과제
  7.1 Phase 2 도메인 시드 [TBD]
  7.2 운영 시작 후 추가 입력 운영 정책 (관리자 화면 가이드)
  7.3 As-Is 시스템 병행 운영 종료 시점

§8 변경 이력 + 관련 문서
```

**원칙**:
- 마이그 색채 제거 (롤백 윈도우·전수 마이그·병행 운영 SQL 등 삭제)
- §5 데이터 정제 섹션 제거 (시드 입력이라 정제 불필요)
- §6 검증 결과는 **IM 단계 "초기 데이터 시드 입력 결과서"의 입력**으로 연계 (PP12-1 v2.2 §1 L58 일관)
- §3.4 자재·제품 시드는 [[7_유니크시스템BOM_26.04.24_분석]] 보완본을 핵심 입력으로 활용

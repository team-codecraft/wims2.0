# 문서코드 · 산출물 작성 규칙

## 파일명 패턴

`WIMS2.0_{문서코드}_{문서명}_v{버전}.{docx|xlsx}`

## 단계별 문서코드

| 단계 | 코드 패턴 | 예시 |
|------|-----------|------|
| PP | PP{활동번호}-{일련번호} | PP11-1 (TFT구성계획서), PP12-1 (테일러링결과서), PP13-1 (개발환경구성결과서) |
| AN | AN{활동번호}-{일련번호} | AN11-2 (인터뷰결과서), AN12-1 (요구사항정의서), AN21 (사이트맵) |
| DE | DE{활동번호}-{일련번호} | DE11-1 (아키텍처설계서), DE24-1 (인터페이스설계서), DE35-1 (업무규칙정의서) |

## 산출물 작성 규칙

- 모든 문서는 한국어로 작성, 버전(v1.0/v1.1 등) 관리, 변경 이력 테이블 포함
- **작성 순서:** md 초안 먼저, 사용자 확인/조율 후 명시적 요청 시에만 docx/xlsx 생성
- 폰트: 맑은 고딕, 색상 체계: 1F4E79(Primary), 2E75B6(Accent), D6E4F0(Header BG)
- 문서 구성: 표지 → 변경이력 → 목차(수동 구성) → 본문 → 승인 테이블
- TOC 는 docx-js 자동 생성이 깨지므로 수동으로 구성
- 생성된 산출물은 `산출물/{단계별 폴더}/` 에 저장

## 산출물 생성 워크플로우

- 생성 스크립트는 `scripts/generate_*.js` 형태로 작성
- docx 는 `docx-js` (Node.js), xlsx 는 `exceljs` 사용
- 패키지 매니저는 **yarn** (npm 금지)
- 실행: `node scripts/<script>.js`
- 산출 경로는 스크립트 내 하드코딩, 파일명 규칙은 위 문서코드 규칙을 따름

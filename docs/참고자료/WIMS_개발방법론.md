**WIMS 시스템 개선**

**SW 개발방법론**

특허청 SW 개발방법론 기반 프로젝트 테일러링

적용 프로젝트 : WIMS (Window & Curtain Wall Information Management
System)

개 발 기 간 : 8개월 (Phase 1: M1\~M3 / Phase 2: M4\~M8)

문 서 버 전 : v1.0 (초안)

참 조 기 준 : 특허청 SW 개발방법론 (객체지향 CBD 기반, ISO/IEC 12207
준용)

**1. 개요**

**1.1 목적**

본 문서는 WIMS 시스템 개선 프로젝트의 개발 방법론을 정의합니다. 특허청
SW 개발방법론(객체지향 CBD 기반, ISO/IEC 12207 준용)을 참조 기준으로
삼고, WIMS 프로젝트의 규모·성격·일정에 맞게 테일러링(Tailoring)하여
적용합니다.

+-----------------------------------------------------------------------+
| **■ 방법론 적용 목적**                                                |
|                                                                       |
| • 개발 단계별 프로세스와 산출물을 표준화하여 개발 생산성 및 시스템    |
| 품질 향상                                                             |
|                                                                       |
| • 기존 시스템 개선 개발의 특성을 반영하여 과도한 문서화 부담 없이     |
| 핵심 품질 확보                                                        |
|                                                                       |
| • Phase 1(BOM+MES 연동) / Phase 2(전체 시스템) 순차 오픈 일정에 맞는  |
| 단계별 산출물 관리                                                    |
|                                                                       |
| • MES DB 직접 연동 등 외부 시스템 연계 품질 검증 체계 수립            |
+-----------------------------------------------------------------------+

**1.2 참조 기준**

  ---------------------- ------------------------------------------------
  **구분**               **내용**

  **참조 방법론**        특허청 소프트웨어(SW) 개발방법론 (객체지향 CBD
                         기반)

  **국제 표준**          ISO/IEC 12207 (소프트웨어 수명주기 프로세스)

  **프로세스 구조**      단계(Phase) 7단계 → 활동(Activity) → 작업(Task)
                         → 산출물(Output)

  **테일러링 방침**      기존 시스템 개선 개발 + 중소 규모 사업 기준
                         테일러링 적용 (필수 산출물 중심 간소화)
  ---------------------- ------------------------------------------------

**1.3 적용 범위**

-   WIMS 시스템 전면 개선 개발 (Phase 1 + Phase 2 전체 범위)

-   ① 제품 관리 시스템 (BOM / 자재 / 공정 / MES 연동)

    -   ② 견적 설계 시스템 (CAD 웹뷰어 / 도면 매핑 / 견적 산출)

    -   ③ 발주 관리 시스템 (계약 확정 / 최적 절단 / 발주서)

    -   ④ 제조 관리 시스템 (제작도 산출 / 절단 지시서 / 추가 발주)

**2. 방법론 테일러링 (Tailoring)**

**2.1 테일러링 기준**

특허청 방법론은 대규모 공공 정보화 사업 기준으로 설계되어 있으므로, WIMS
프로젝트의 아래 특성을 반영하여 산출물 및 절차를 간소화합니다.

+-----------------------------------------------------------------------+
| **■ 테일러링 적용 근거**                                              |
|                                                                       |
| ① 기존 시스템 개선 개발 : 신규 구축이 아닌 레거시 개선으로 As-Is 분석 |
| 범위 한정                                                             |
|                                                                       |
| ② 중소 규모 사업 : 8개월, 소규모 TFT → 과도한 산출물보다 핵심 품질    |
| 산출물 중심 운영                                                      |
|                                                                       |
| ③ 2-Phase 순차 오픈 : Phase 1 (M3) 단독 오픈 시점에 별도 테스트·전개  |
| 프로세스 적용                                                         |
|                                                                       |
| ④ MES DB 직접 연동 : REST API 개발 없이 DB 연동 방식 → 인터페이스     |
| 설계 간소화                                                           |
|                                                                       |
| ⑤ 애자일 요소 병행 : 구현 단계에서 2주 단위 스프린트 적용, 주요 기능  |
| 단위 데모 실시                                                        |
+-----------------------------------------------------------------------+

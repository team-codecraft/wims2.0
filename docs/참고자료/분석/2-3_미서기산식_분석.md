## 1. 개요

### 1.1 파일의 역할

이 파일은 유니크시스템 미서기창호 **BOM 자동 산출 엔진의 원장(原帳)** 이다. 창호 전체 치수(W×H), 창호 형태(분할 수), 유리 분할 구조를 입력하면 각 부재의 **절단 치수·수량·소요 자재코드**를 결정하는 산식을 시스템별·형태별로 망라한 산식표이다.

- **짝 문서 관계**: 이 산식 → (치수 대입) → `2-2 제작지시서` (실제 절단 리스트)
- **도면 근거**: `2-1.미서기_도면.pdf` 에 각 부재 치수 기호(W, H, 후렘 두께 등) 정의

### 1.2 시트 구성 (60개 시트)

시트 번호 = 창호 품목 × 창호 형태의 조합. 총 **5개 품목계열 × 12개 형태**로 구성된다.

#### 창호 품목계열 (A열 창호품목명)

| 품목코드 | 창호품목명 | 계열 | 프로파일 | 시트 범위 |
|---------|-----------|------|---------|---------|
| 160-우수 | 24년-160AL-우수1 | 우수 | 160 AL | 1~12 |
| 160-마스 | 21년-160AL-마스1 | 마스터 | 160 AL | 13~24 |
| 225-우수 | 24년-225AL-우수1 | 우수 | 225 AL | 25~36 |
| 225-마스 | 19년-225AL-마스1 | 마스터 | 225 AL | 37~48 |
| 226-마스 | 20년-226AL-마스1 | 마스터 | 226 AL | 49~60 |

> **우수(優秀)** = 고급형(클리핑 후렘+탄성레일, 레일바 별도). **마스(마스터)** = 표준형.

#### 창호 형태 (B열 창호형태명)

창호 형태는 `W분할수 × H분할수 - 편수` 의 조합이다.

| 형태명 | W 분할 | H 분할 | 설명 |
|--------|-------|-------|------|
| W1XH1-정 | 1 | 1 | 단창 (1짝×1단) |
| W1XH2-정 | 1 | 2 | 수직 2단 |
| W1XH3-정 | 1 | 3 | 수직 3단 |
| W2XH1-정 | 2 | 1 | 수평 2짝 |
| W2XH2-정 | 2 | 2 | 2짝×2단 |
| W2XH3-정 | 2 | 3 | 2짝×3단 |
| W3XH1-정 | 3 | 1 | 수평 3짝 |
| W3XH2-정 | 3 | 2 | 3짝×2단 |
| W4XH1-정 | 4 | 1 | 수평 4짝 |
| W4XH2-정 | 4 | 2 | 4짝×2단 |
| W1XH1-3편 | 1 | 1 | 단창, 3편 구조 (중간짝 별도) |
| W1XH2-3편 | 1 | 2 | 수직 2단, 3편 구조 |

> **3편**: W1X 형태에서 가운데 문짝(W-W1-W1 부분)이 별도 규격으로 산출되는 구조. 좌우 문짝과 중간 문짝의 치수가 다름.

### 1.3 시트 내 컬럼 구조

모든 60개 시트가 동일한 11컬럼 구조를 공유한다.

| 열 | 컬럼명 | 내용 |
|----|--------|------|
| A | 창호품목명 | 창호 계열 식별자 (예: `24년-160AL-우수1`) |
| B | 창호형태명 | 분할 형태 (예: `W2XH2-정`) |
| C | 산식구분 | `공통` / `외창` / `내외` — 적용 창 구분 |
| D | 자재코드 | 부재 자재코드 (예: `UNI-A160-101`) |
| E | 자재명칭 | 부재 명칭 (예: `160 -H- AL프레임`) |
| F | 소요구분 | `후렘` / `문짝` / `방충` — 부재 카테고리 |
| G | 위치 | 적용 치수 방향 (`W`, `H`, `W1`, `H1`, `H2` 등) |
| H | 위치2 | 조건 분기 메모 (예: `900이하`) |
| I | 수량 | 수량 (숫자 또는 IIF 조건식) |
| J | 산식1 | **1차 치수 산식** (부재 절단 길이, mm 단위) |
| K | 산식2 | **2차 치수 산식** (유리류의 세로 치수 또는 보조 길이) |

## 2. 입력 변수

산식 전반에서 사용되는 입력 파라미터. 이 값들이 J, K, I열 산식에 직접 대입된다.

| 변수명 | 의미 | 단위 | 비고 |
|--------|------|------|------|
| `W` | 창호 전체 너비 (실측 개구부 폭) | mm | 최상위 입력값 |
| `H` | 창호 전체 높이 (실측 개구부 높이) | mm | 최상위 입력값 |
| `W1` | 3편 구조 시 좌우 문짝 1개의 너비 | mm | 3편 시트에서만 사용 |
| `H1` | 수직 2단·3단 시 하단 창 높이 | mm | XH2, XH3 시트에서 사용 |
| `H2` | 수직 3단 시 중단 창 높이 | mm | XH3 시트에서 사용 |

### 입력 변수 주요 범위 (산식 추정)

- **H >= 900 조건**: 방충망 후렘 수량 결정 임계값 (H≥900이면 상하단 각 1개 추가)
- `W1`: 3편 형태에서 `W - 2*W1` = 중간 문짝 너비 (W1은 사용자 지정)

## 4. 핵심 수식 — 셀 인용 및 의미 설명

모든 산식은 셀 값으로 **텍스트(문자열) 형태**로 저장되어 있으며, 엑셀 수식이 아닌 **자체 산식 언어**이다. 시스템에서 W, H 등을 치환·계산해 사용한다.

### 4.1 후렘 (외곽 프레임) 산식

**시트 1. 160-우수-1X1, 행 2~3**

```
D2: UNI-A160-101   G2: W   I2: 2   J2: W
D3: UNI-A160-101   G3: H   I3: 2   J3: H
```

- 상하 수평 프레임 2개: 절단 길이 = `W` (창호 전체 폭 그대로)
- 좌우 수직 프레임 2개: 절단 길이 = `H` (창호 전체 높이 그대로)

**시트 1. 160-우수-1X1, 행 4 (레일바)**

```
D4: UNI-PA160-401   G4: W   I4: 3   J4: W-73-73
```

- 레일바 3개: 절단 길이 = `W - 73 - 73`
- 의미: 창호 폭에서 좌·우 후렘 단부 여유 각 73mm씩 빼서 레일바 수납 홈에 맞춤

**시트 5. 160-우수-2X2, 행 4 (2짝 분할 레일바)**

```
D4: UNI-PA160-401   G4: W   I4: 12   J4: (W-73-73-132)/2
```

- 레일바 12개: 절단 길이 = `(W - 73 - 73 - 132) / 2`
- 의미: 2짝 분할 시 중간 분리재(132mm) 고려 후 각 짝별 레일바 길이 산출

**W분할별 레일바 산식 패턴**

| 분할 | 산식 | 의미 |
|------|------|------|
| 1짝 | `W-73-73` | 전체 폭 - 양단 여유 |
| 2짝 | `(W-73-73-132)/2` | 중간재 132mm 1개 제거 후 2등분 |
| 3짝 | `(W-73-73-132-132)/3` | 중간재 132mm 2개 제거 후 3등분 |
| 4짝 | `(W-73-73-132-132-132)/4` | 중간재 132mm 3개 제거 후 4등분 |

### 4.2 문짝 너비 산식

**시트 1. 160-우수-1X1, 행 5 (160 우수 단창 문짝 W)**

```
D5: YJN-SL160A-401   G5: W   I5: 4   J5: (W/2)-2-2+3
```

- 4개(내·외 문짝 각 2매) × 절단 길이 = `(W/2) - 2 - 2 + 3`
- 의미: W를 2짝으로 나눈 뒤, 레일 겹침 여유(-2, -2)와 클리핑 여분(+3) 적용

**시트 25. 225-우수-1X1, 행 5 (225 우수 단창 문짝 W)**

```
D5: UNI-A225-401   G5: W   I5: 4   J5: (W/2)-3-3
```

- 절단 길이 = `(W/2) - 3 - 3` (225 계열은 여유값 3mm)

**W분할별 문짝 너비 산식 패턴 (160 우수 계열)**

| 분할 | 산식 | 수량 |
|------|------|------|
| 1짝 | `(W/2)-2-2+3` | 4 |
| 2짝 | `(W/4)-2+1+3` | 16 |
| 3짝 | `(W/6)-3-0+1` | — |
| 4짝 | `(W/8)+1+3` | — |

### 4.3 문짝 높이 산식

**시트 1. 160-우수-1X1, 행 6~7**

```
D6: YJN-SL160A-401   G6: H   I6: 4   J6: H-37-37-1
D7: UNI-P160-MC      G7: H   I7: 2   J7: H-37-37-2-1
```

- 문짝 세로 프로파일: `H - 37 - 37 - 1`
  - 의미: 창호 전체 높이에서 상·하 후렘 두께 각 37mm와 유격 1mm 차감
- 고리캡(짧은 마감재): `H - 37 - 37 - 2 - 1`
  - 의미: 문짝 높이에서 추가 여유 2mm 차감

**수직 2단(XH2) 시트의 문짝 높이 패턴 (시트 2)**

```
G7: H1   J7: H1-37-30-1    (하단 창, H1 적용)
G8: H    J8: H-H1-37-30-1  (상단 창, H-H1 적용)
```

- H1: 하단 단위 창 높이 / (H-H1): 상단 단위 창 높이

**수직 3단(XH3) 패턴**

```
H1-37-30-1      (하단)
H2-37-30-1      (중단)
H-H1-H2-37-30-1 (상단)
```

### 4.4 방충망 후렘 산식 (IIF 조건)

**시트 1. 160-우수-1X1, 행 8~10**

```
D8: SC-1911   G8: W   I8: 2       J8: (W/2)+3+3
D9: SC-1911   G9: H   I9: IIF(H>=900,2,0)    J9: H-35.5-35.5
D10: SC-1911  G10: H  I10: IIF(H<900,2,0)    J10: H-35.5-35.5-2
```

- 방충망 W 방향: `(W/2)+3+3` — 각 짝 너비 + 양단 3mm 여유
- **조건 분기**: H≥900mm이면 상하 각 1개(수량 2), H<900이면 수량 2이나 2mm 짧게 절단
  - H10의 `H-35.5-35.5-2`: 낮은 창호에서 방충망 내부 설치 시 하부 여유 2mm 추가 감소

**IIF 패턴 전체 (수량 결정)**

```
IIF(H>=900, N, 0)      — H 단창, 900 이상이면 N개, 아니면 0
IIF(H1>=900, N, 0)     — 하단 창(H1), 900 이상
IIF((H-H1)>=900, N, 0) — 상단 창(H-H1), 900 이상
IIF((H-H1-H2)>=900, N, 0) — 3단 최상단, 900 이상
```

N값 범위: 2, 4, 6, 8 (분할 수에 비례)

### 4.5 유리 치수 산식 (K열 활용)

**시트 1. 160-우수-1X1, 행 11**

```
D11: ZZ-ZZ100   G11: W   I11: 2
J11: ((W/2)-2-2)-69.5-69.5+3     (← 유리 너비)
K11: (H-37-37)-69.5-69.5          (← 유리 높이)
```

- J열 = 유리 너비: 문짝 너비에서 유리 삽입 홈 깊이(글레이징 비드) 각 69.5mm 차감
- K열 = 유리 높이: 창호 높이에서 상하 후렘 37mm 및 글레이징 비드 69.5mm씩 차감

**225 계열 유리 치수 (시트 25)**

```
J11: (W/2)-3-3-54.5-54.5
K11: H-38-38-54.5-54.5
```

- 225 계열: 문짝 홈 여유 3mm, 글레이징 비드 54.5mm (160의 69.5mm와 차이)
- 후렘 두께: 38mm (160의 37mm와 차이)

**마스터형 유리 치수 (시트 37. 225-마스-1X1)**

```
J10: (W/2)-3.5-3.5-54.5-54.5
K10: H-99-99+15-2
```

- 마스터형: 문짝 홈 3.5mm, K열 `H-99-99+15-2` → 상하 99mm 누적 차감 후 +15mm 교정

### 4.6 모헤어(씰) 길이 산식

**시트 1. 160-우수-1X1, 행 16~18**

```
D16: 04-0002   F16: 문짝   I16: 1   J16: ((W*4)+(H*4))
D17: 04-0002   F17: 방충   I17: 1   J17: H
D18: 04-0003   F18: 후렘   I18: 1   J18: W*2
```

- 문짝용 6.7×5mm 모헤어 총 길이: `(W*4)+(H*4)` — 4면 × (W+H)
- 방충용 모헤어: `H` — 수직 방향만
- 후렘용 6.7×7mm 모헤어 총 길이: `W*2` — 상하 프레임 2개

**다분할 문짝 모헤어 패턴**

| 창호 형태 | 문짝 모헤어 산식 | 후렘 모헤어 산식 |
|---------|----------------|----------------|
| W1XH1-정 | `(W*4)+(H*4)` | `W*2` |
| W1XH2-정 | `(W*8)+(H*4)` | `W*4` |
| W2XH1-정 | `(W*4)+(H*8)` | `W*4` |
| W2XH2-정 | `(W*8)+(H*8)` | `W*4` |

### 4.7 탄성스프링 수량 산식

**시트 1. 160-우수-1X1, 행 20**

```
D20: 02-0051   F20: 후렘   I20: 1   J20: (W/250)*2
```

- 탄성스프링 개수: 창호 폭 250mm마다 2개씩 추가
- 시트별 패턴: `(W/250)*2`, `(W/250)*4`, `((W/2)/250)*8` 등 분할에 비례

### 4.8 방충망O형가스켓 둘레 산식

**시트 1. 160-우수-1X1, 행 27**

```
D27: 04-0018   F27: 방충   I27: 1   J27: ((W/2)*2)+(H*2)
```

- 방충망 가스켓 총 길이 = `(W/N × 문짝수) + (H × 2)` → 창짝 둘레 총합

### 4.9 3편(W1 분할) 특수 산식

**시트 11. 160-우수-1X1-3편, 행 5~7**

```
G5: W1    I5: 4   J5: W1-2-2+3              (좌우 소창짝 W 방향)
G6: W     I6: 2   J6: W-W1-W1+33+33         (중간 대창짝 W 방향)
G7: H     I7: 6   J7: H-37-37-1             (전체 문짝 H 방향, 6개)
```

- W1: 좌우 단부 문짝의 너비 (입력값)
- `W-W1-W1+33+33`: 중간 문짝 너비 = 전체 폭 - 좌우 문짝 폭 + 클리핑 여분 각 33mm
- 문짝유리 (행 12~13):
  ```
  J12: (W1-2-2)-69.5-69.5+3    K12: (H-37-37)-69.5-69.5  (좌우 유리)
  J13: (W-W1-W1+33+33)-69.5-69.5  K13: (H-37-37)-69.5-69.5  (중간 유리)
  ```

## 5. 부재별 계산 로직 요약

### 5.1 외곽 후렘 (H형 프레임)

```
절단 치수(W방향) = W           수량 = 2
절단 치수(H방향) = H           수량 = 2
```

모든 품목·형태에서 동일. W, H 그대로 절단.

### 5.2 중간 프레임 (XH2 이상)

```
수평 중간 프레임 = W - 47 - 47         (160 계열)
수직 중간 프레임(H1 구간) = H1 - 47 - 40
수직 중간 프레임(H-H1 구간) = H-H1 - 47 - 40
```

중간 프레임 연결재(02-0072 등) 고정 수량은 분할 수에 따라 결정.

### 5.3 레일바 (탄성승강용)

```
레일바 길이 = (W - 73×2 - 132×(W분할-1)) / W분할
```

- 160 우수 단창: `W-73-73` × 3개
- W 분할이 늘수록 개당 길이 줄고 총 수량은 `3×W분할×H분할` 비례

### 5.4 문짝 프로파일

```
너비 = (W / W분할수) - 양단여유 + 클리핑여분
높이 = (단위 높이) - 상하후렘두께 - 여유
```

계열별 여유값:
| 계열 | 너비 여유(양쪽) | 높이 상단 | 높이 하단 |
|------|---------------|---------|---------|
| 160 우수 | -2, -2, +3 | -37 | -37 |
| 225 우수 | -3, -3 | -38 | -38 |
| 225 마스 | -3.5, -3.5 | -47, +10 | -47, +10 |
| 226 마스 | 0 (반값 그대로) | -31 | -31 |

### 5.5 유리 치수

```
유리 너비 = 문짝너비 - 글레이징비드×2
유리 높이 = 단위높이 - 상하후렘두께 - 글레이징비드×2
```

계열별 글레이징 비드 값:
| 계열 | 비드(양쪽 합) |
|------|------------|
| 160 | 69.5×2 = 139mm |
| 225 우수 | 54.5×2 = 109mm |
| 225/226 마스 | 54.5×2 = 109mm (H방향은 상이) |

### 5.6 방충망 후렘

```
너비 = (W / W분할수) + 단부여유×2
높이 = 단위높이 - 상하여유
  — H≥900: 여유 작음 (레일 내부 삽입)
  — H<900: 여유 2mm 추가 차감 (별도 취부)
```

### 5.7 씰(모헤어) 총 길이

```
문짝 6.7×5mm = (W × 문짝변수) + (H × 문짝변수)   — 전체 둘레 합산
후렘 6.7×7mm = W × N                              — 상하 방향만
방충 6.7×5mm = H (수직 방향)
```

## 8. WIMS 관점 시사점

### 8.1 룰 엔진 모델링 방향

**1. 다차원 룩업 키**

각 산식 행은 **(창호품목, 창호형태, 소요구분, 자재코드, 위치)** 5개 키로 유일하게 식별된다. WIMS 룰 엔진의 기본 데이터 구조:

```
BomRule {
  productCode: String    // 창호품목명 (5개)
  formCode: String       // 창호형태명 (12개)
  formulaCategory: String // 산식구분 (공통/외창/내외)
  materialCode: String   // 자재코드
  direction: String      // 위치 (W/H/W1/H1/H2)
  qtyFormula: String     // 수량 산식 (숫자 or IIF 식)
  dimFormula1: String    // 절단 치수 산식1 (J열)
  dimFormula2: String    // 절단 치수 산식2 (K열, 유리 세로)
}
```

**2. 파라미터화 수준**

- **직접 대입형** (90%): W, H, H1, H2, W1에 숫자 대입 후 산술 연산
- **조건 분기형** (10%): `IIF(H>=900, N, 0)` — 이진 조건, 임계값 900mm 하드코딩
- **비례 수량형**: 탄성스프링 `(W/250)*N` — W에 선형 비례

→ 복잡한 중첩 조건 없음. 단순 사칙연산 + 이진 IIF로 전체 커버 가능.

**3. 시트 → 룰 행 매핑 전략**

60개 시트를 1개 테이블로 통합 후 (창호품목, 창호형태)를 컬럼으로 관리. 총 행 수:
- 단창(W1XH1): 23~31행 × 5품목 = ~145행
- 전체 60시트 합산: 약 1,900~2,000행 규모 (중복 제거 전)

**4. 중간 프레임 수 결정 로직**

중간 프레임 개수는 `W분할-1`, `H분할-1`로 결정. 연결재 수량도 이에 비례. → 산식 내 하드코딩이 아닌 **W분할수·H분할수를 파라미터로 받아 연결재 수량 자동 계산** 가능.

### 8.2 구현 우선순위

| 우선순위 | 항목 | 근거 |
|---------|------|------|
| P1 | 160 우수 W1XH1-정 룰 구현 | 가장 단순, 검증 기준 |
| P1 | IIF 조건 파서 | 방충망 수량 분기 |
| P2 | W/H 분할 수 파라미터화 | 60개 시트 통합 전제 |
| P2 | 225 계열 여유값 분리 | 계열별 상수 파라미터화 |
| P3 | 3편 형태 W1 입력 | 추가 입력 변수 |
| P3 | 유리 치수 자동 산출 | ZZ-ZZ100 참고용 활용 |

### 8.3 검증 데이터셋 후보

`2-2.미서기_제작지시서(절단및 수량리스트).xlsx` 가 실제 W, H 값을 대입한 결과물이므로, 이 파일에서:
1. 실제 입력 W, H 값 추출
2. 본 산식으로 계산된 이론값과 제작지시서의 실제값 비교
3. 오차 허용 범위(보통 ±1mm) 내 일치 여부로 룰 엔진 검증

### 8.4 주의사항

- **산식 텍스트 부동소수점**: 35.5, 36.5, 69.5 등 소수점 상수 → DB 저장 시 Decimal 타입 사용
- **마스터형 vs 우수형 여유값 차이**: 동일 자재코드라도 계열에 따라 절단 상수가 다름 (예: 225마스 H방향 `-47*2+10*2-2` vs 225우수 `-38-38`)
- **900mm 임계값**: 현장에서 조정 가능한 값일 수 있음 → WIMS 파라미터로 노출 권장
- **H열 위치2 컬럼**: `900이하` 메모만 있으며 실제 조건은 I열 IIF에 내포 → DB 설계 시 remark 컬럼으로 처리

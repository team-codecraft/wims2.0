## 1. 개요

### 문서 목적

유니크시스템의 미서기(미닫이) 창호 제품에 대해 **알루미늄 원자재 절단 길이·수량**과 **부자재 수량**을 제품 타입별로 사전 정의한 제작지시서(BOM 전개 + 절단 리스트). 실제 발주 시 창호 치수(W×H)와 Set 수를 입력하면 이 양식에 따라 절단 리스트가 완성된다.

### 작성 기준 (상단 노트)

| 항목 | 내용 |
|------|------|
| 기준 치수 | 1칸(1X1) = W 1000 × H 1000 mm |
| 3편(三枚) 예외 | 1X1-3편은 W1 = 300 mm |
| 시리즈-연도 | 160 우수 → 24년형 / 160 마스 → 21년형 / 225 우수 → 24년형 / 225 마스 → 19년형 / 226 마스 → 20년형 |

## 4. 데이터 구조 (컬럼 정의)

### 4.1 제품 헤더 블록 (3행 구성)

각 제품 타입 시작부에 3행의 헤더가 위치한다.

| 행 | A열 | B열 | D열 | F열 | G~L열 |
|----|-----|-----|-----|-----|-------|
| 헤더1 | (시리즈-타입) 예: `(160-우수-1X1)` | 연도-규격코드 | 타입코드 (W1XH1-정) | Set 수 | W1/W2/W3 값 |
| 헤더2 | 기준치수 (1000×1000) | | 색상(외/내) | | H1/H2/H3 값 |
| 헤더3 | 동호수 | | 위치 | | |

### 4.2 부재 절단 리스트 컬럼 구조

| 열 | 위치 | 데이터 |
|----|------|--------|
| D | 4 | **그룹** — `공통` / `외창` / `내창` |
| E | 5 | **부품코드** (예: `UNI-A160-101`) |
| F | 6 | **부품명** (예: `160 -H- AL프레임`) |
| I | 9 | **치수 방향 키** — `W` / `H` / `W1` / `H1` / `H2` 등 |
| J | 10 | **절단 길이 (mm)** |
| K | 11 | 구분자 `×` (곱하기) |
| L | 12 | **수량 (개)** |

> 모헤어(선형 자재)·방충망(면적 자재)는 mm 또는 mm×배수 형태로 표기.

### 4.3 병합셀 패턴

- `A1:L1` — 문서 제목 전체 병합
- `D{n}:L{n}` — 각 제품 그룹 구분 구역 경계 행 병합 (그룹 헤더 하단 공백행)
- `P1:U9` — 우측 메모 영역 (작성 기준 노트)

## 8. WIMS MF 관점 시사점

### 8.1 제작지시서 자동 생성 로직 후보

1. **입력 파라미터 (트리거)**
   - 제품 시리즈 코드 (`160-우수` / `160-마스` / `225-우수` / `225-마스` / `226-마스`)
   - 레이아웃 타입 (`WmXHn[-3편]`)
   - 창호 치수 W, W1~W3, H, H1~H3 (mm)
   - 발주 Set 수

2. **산출 로직 (2-3 산식.xlsx 기반 추정)**
   - 각 부재별 절단 길이 = f(W, H, W1, H1, H2, H3, 시리즈 옵셋 상수)
   - 수량 = 레이아웃 타입별 고정 배수 × Set 수
   - 3편 여부에 따라 문짝 W 분할 공식 분기

3. **출력 (제작지시서)**
   - 부재코드·부품명·치수방향·절단길이·수량 리스트 (절단 지시)
   - 부자재 수량 리스트 (계수 지시)
   - 그룹 구분 (공통/외창/내창)

### 8.2 필요 엔티티/속성 (MF 서브시스템)

```
ProductSeries (제품 시리즈)
  ├─ seriesCode       VARCHAR  -- '160-우수', '225-마스' 등
  ├─ modelYear        YEAR     -- 연도 (24년, 21년 등)
  ├─ profileFamily    VARCHAR  -- 'UNI-A160', 'YJN-SL160A', 'UNI-A225', 'UNI-M226'

LayoutType (레이아웃 타입)
  ├─ layoutCode       VARCHAR  -- 'W1XH1-정', 'W1XH1-3편' 등
  ├─ gridW            INT      -- 가로 칸 수
  ├─ gridH            INT      -- 세로 단 수
  ├─ is3Piece         BOOLEAN  -- 3편(三枚) 여부

CuttingBOM (절단 BOM 마스터)
  ├─ seriesCode       FK → ProductSeries
  ├─ layoutCode       FK → LayoutType
  ├─ itemCode         VARCHAR  -- 부품코드
  ├─ itemName         VARCHAR
  ├─ group            ENUM('공통', '외창', '내창')
  ├─ dimKey           VARCHAR  -- 'W', 'H', 'W1', 'H1', 'H2' 등
  ├─ formula          TEXT     -- 절단 길이 공식 (2-3산식 기반)
  ├─ qtyPerSet        DECIMAL  -- Set당 수량

WorkOrder (제작지시서 인스턴스)
  ├─ orderId
  ├─ clientName
  ├─ fieldName
  ├─ contractDate     DATE
  ├─ frameDelivery    DATE
  ├─ sashDelivery     DATE
  ├─ screenDelivery   DATE
  ├─ revision         INT      -- 차수

WorkOrderLine (제작지시서 라인)
  ├─ workOrderId      FK → WorkOrder
  ├─ locationCode     VARCHAR  -- 동·호
  ├─ seriesCode
  ├─ layoutCode
  ├─ dimW, dimH       INT      -- 실측 치수(mm)
  ├─ dimW1..W3        INT
  ├─ dimH1..H3        INT
  ├─ setQty           INT
  ├─ colorOuter       VARCHAR
  ├─ colorInner       VARCHAR

CuttingInstruction (절단 지시 출력)
  ├─ workOrderLineId  FK
  ├─ itemCode
  ├─ itemName
  ├─ group
  ├─ dimKey
  ├─ cutLength        INT      -- 계산된 절단 길이(mm)
  ├─ qty              INT
```

### 8.3 주요 설계 고려사항

| 이슈 | 내용 |
|------|------|
| 시리즈별 프레임 코드 분기 | 동일 레이아웃이라도 시리즈에 따라 부품코드 상이 → CuttingBOM을 시리즈×레이아웃 복합키로 관리 |
| 절단 옵셋 상수 관리 | 시리즈마다 옵셋 값 다름 (74mm vs 76mm 등) → ProductSeries에 상수 테이블 별도 필요 |
| 3편 분기 | W1 값이 있을 때 문짝 W 공식이 달라짐 → LayoutType.is3Piece 플래그 + 별도 공식 참조 |
| 단수(H칸) 확장 | H1/H2/H3 분할 시 문짝이 구간별로 H방향 다름 → dimKey 'H', 'H1', 'H2' 각각 별도 CuttingInstruction 행 |
| 부자재 vs 절단재 구분 | 절단 길이 있는 알루미늄 프로파일과 EA 계수 소모품을 동일 BOM 테이블로 통합 관리 (type 구분 컬럼 필요) |
| 방충망 면적 계산 | ZZ-ZZ03 방충망(M)은 mm × mm → 평 환산 (`×1000=0.50평`) → 별도 면적 계산 필드 필요 |
| 전체 시트 vs 개별 시트 | 전체 시트는 출력용 통합본, 개별 시트는 단건 출력 템플릿 → WIMS에서는 개별 시트 구조를 단건 제작지시서 출력 모델로 활용 |

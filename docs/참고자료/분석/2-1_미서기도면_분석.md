### 1.1 도면 구성 방식

각 페이지는 동일 양식을 반복한다:

- **상단 헤더**: 현장명, 색상, 유리 사양(내부/외부), 품목명, 방충망 유무, 포장 방식, 작성일, 특이사항
- **우측 상단**: 창호 평면도(입면 개략 스케치) — 짝 수와 구성(편·연창 등) 시각 표기
- **하단 표**: 기호(품번), 수량, W, W1, H, H1, 위치번호, 방충, 비고

### 1.2 창호 유형 분류 체계 (품번 코드 해석)

기호 패턴: `{계열폭}-{창문타입}-{짝W}X{단H}[-3편]`

| 코드 예시 | 해석 |
|---|---|
| `160-우수-1X1` | 160mm 시리즈, 우수(우측 열림), 가로 1짝, 높이 1단 |
| `160-우수-2X2` | 160mm, 우수, 가로 2짝, 높이 2단 |
| `225-마스-3X2` | 225mm, 마스터(양측 열림), 가로 3짝, 높이 2단 |
| `226-마스-1X1-3` | 226mm, 마스터, 가로 1짝·높이 1단, 3편창(고정+열림+고정) |

### 2.2 치수 조합 매트릭스 (각 그룹 공통 패턴)

| 짝W 기호 | W 치수 | 짝H 기호 | H 치수 | 비고 |
|---|---|---|---|---|
| 1X | W = 1,000 | X1 | H = 1,000 | — |
| 2X | W = 2,000 | X2 | H = 2,000, H1 = 1,000 | 중간 프레임 추가 |
| 3X | W = 3,000 | X3 | H = 3,000, H1 = 1,000 | — |
| 4X | W = 4,000 | — | — | — |
| 3편 | W1 = 1,000, W = 전체 | X1-3 | H = 1,000, 300 | 3편창(고정+미서기+고정) |

> W1 = W / 짝수, H1 = 중간 높이 (연창인 경우)

### 7.3 산식 패턴 분류 (BOM Rule 설계 관점)

용어사전의 `BOM_RULE (옵션별규칙)` 대응:

| 규칙 유형 | 조건 | 결과 |
|---|---|---|
| 레일바 수량 규칙 | 단창(X1) → 3EA, 연창(X2) → 6EA | 단/연창 구분 |
| 중간 프레임 규칙 | X2 이상 → 수평 중간 프레임 추가 | H1 존재 여부 |

### 7.4 ES/MF 서브시스템 연계 시사점

**ES(견적설계) 관점:**
- W, H, frame_type, vent_type, vent_w_count, vent_h_count, is_3vent, has_screen, glass_type 6~8개 변수만으로 전 자재의 소요량 산출 가능
- 산식이 각 시트에 완전히 기술되어 있으므로 WIMS의 `BOM_RULE` 엔티티로 직접 이식 가능
- 유리(ZZ-ZZ100)는 W×H 2차원 치수 산식 — 별도 유리 견적 모듈 필요

**MF(제조관리) 관점:**
- 60개 시트가 각각 하나의 `Resolved BOM` 템플릿에 대응
- 절단 방향(W방향/H방향)과 위치(H01/H02/W01 등)가 이미 구분되어 있음 → MBOM 공정 노드와 직접 매핑 가능
- `산식구분: 공통/외창/내외`가 MBOM의 `is_phantom` 및 `workCenter` 분기와 연결
- 방충망 조건 분기(H≥900)는 `BOM_RULE` 의 조건식 필드에 등재 필요

**데이터 정합성 이슈:**
- 도면 페이지 표기가 48페이지에서 재시작(`1/48` → `49/60`)하는 것은 226mm 계열이 별도 배치로 추가된 것. WIMS에서 제품군을 독립된 `standardBomId`로 관리할 것을 권장.
- W1, H1이 도면에서는 파생변수로만 표기되나 3편창에서는 W1이 입력값(소짝 폭)으로도 쓰임 → ES 입력 폼에서 별도 처리 필요.
- 산식 구분 `내외`(225계열)는 내창·외창 동일 자재를 공유함을 의미 — 160계열의 `외창`(외창 전용)과 다른 개념임에 주의.

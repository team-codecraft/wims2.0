## 3. 계열(시리즈) / 시스템 분류 체계

### 3-1. 2축 분류 체계

```
커튼월 계열
├── 마스(MAS) — 일반 외장 커튼월 (노출 캡바 또는 히든바)
│   ├── 삼중유리
│   │   ├── 특대 (165계열, D=165mm)  → UNI-D-17
│   │   ├── 대   (135계열, D=135mm)  → UNI-D-19
│   │   ├── 중   (105계열, D=105mm)  → UNI-D-21
│   │   └── 소   ( 75계열, D= 75mm)  → UNI-D-23
│   └── 복층유리
│       ├── 특대 (165계열)            → UNI-D-18
│       ├── 특대2(173계열, 신규)       → UNI-D-18-1
│       ├── 대   (135계열)            → UNI-D-20
│       ├── 중   (105계열)            → UNI-D-22
│       └── 소   ( 75계열)            → UNI-D-24
└── 우수(WS) — 우수 처리형 커튼월
    ├── 삼중유리
    │   ├── 특대 (165계열)            → UNI-D-25
    │   ├── 대   (135계열)            → UNI-D-27
    │   ├── 중   (105계열)            → UNI-D-29
    │   └── 소   ( 75계열)            → UNI-D-31
    └── 복층유리
        ├── 특대 (165계열)            → UNI-D-26
        ├── 대   (135계열)            → UNI-D-28
        ├── 중   (105계열)            → UNI-D-30
        └── 소   ( 75계열)            → UNI-D-32
```

### 3-2. 주요 계열 치수 (멀리언 폭 기준)

| 계열 코드 | 멀리언 외폭(D) | 트랜섬 외폭 | 적용 유리 | 비고 |
|-----------|--------------|------------|----------|------|
| 165계열 | 165mm | 165mm | 삼중 또는 복층 | 특대 스팬 |
| 173계열 | 173mm | 173mm | 복층 | 신규(2025.08) |
| 135계열 | 135mm | 135mm | 삼중 또는 복층 | 대 스팬 |
| 105계열 | 105mm | 105mm | 삼중 또는 복층 | 중 스팬 |
| 75계열 | 75mm | 75mm | 삼중 또는 복층 | 소 스팬 |

### 3-3. 유리 타입별 치수 차이

- **삼중유리**: 캡바 단열재 두께 ~60.3mm, 히든바 단열재 두께 ~50.8mm
- **복층유리**: 캡바 단열재 두께 ~42.3mm, 히든바 단열재 두께 ~33.3mm
- **설치 방식**: 캡형(노출 캡바) / 히든형(은폐 히든바) 두 가지 선택

### 7-3. 데이터 이관 전략

1. **계층 구조 매핑**: 커튼월 시스템 → 계열(마스/우수) → 크기(특대/대/중/소) → 유리종류 → 설치방식(캡/히든)의 5단계 분류 체계를 WIMS 제품 분류 트리에 그대로 반영

2. **표준 부재 vs 옵션 부재 구분**: 몸통바의 두께별 변형(1.2t~4.0t)은 옵션 속성으로 처리, 기준 두께(2.0t 또는 2.5t)를 기본값으로 설정

3. **공통 부재 처리**: `연결재(DHS-CN, DHS-CN12)`, `멀리언/트랜섬캡(UNI-CW-C1)`, `마감캡 단열재(UNI-CW-MG-PA)`, `히든바 마감캡(UNI-CW-TMG/DMG)` 등은 계열 구분 없이 공통 사용 — 부재 마스터에서 `series_code = ALL` 처리

4. **금형사 다중 공급**: 동일 부재코드가 대양/성진/삼산 중 어느 금형사에서도 조달 가능한 경우 공급업체 테이블(`vendor_item`) 별도 관리 필요

5. **단열재(PA) 코드 미부여 문제**: PA 단열재 일부는 내부 코드가 없이 명칭+중량만 표기됨 — WIMS 이관 시 `P.A-CW3-모(마)` 형태의 임시코드를 공식코드로 전환하거나, 품목 등록 시 부재코드 신규 발급 필요

6. **가격 정보 연계**: 단가(원/m, 원/EA)가 일부 PA·가스켓 품목에만 표기됨 — 구매 단가는 별도 구매마스터(purchase_price)로 관리하고 다이스북 데이터는 설계 BOM 참조용 기술 마스터로 활용

7. **도면번호(UNI-D-XX) 연계**: 각 도면번호는 설계 기준 문서 역할 — 부재마스터에 `drawing_no` 필드를 두어 변경 이력 추적 가능하게 구성

### 7-4. BOM 전개 시 주의사항

- **마스 vs 우수 구분**: 동일 계열·크기·유리종류라도 마스(MAS)와 우수(WS)는 일부 부재가 다름 (주로 캡바 단열재 규격 차이). BOM 템플릿을 계열별로 분리 관리 필요.
- **캡형/히든형 분기**: 단열재, 유리받침, 마감캡이 캡형과 히든형에서 상이 — 제품 구성(Config) 옵션으로 분기 처리
- **트랜섬/멀리언 공용**: 몸통바 프로파일은 멀리언(수직)과 트랜섬(수평) 모두 동일 프로파일 사용 — 절단 길이만 다름

# CX2 — 용어사전 v1.3 신규 개념 × 6개 후속 문서 커버리지 검사

> [!abstract] 요약
> - **검사 대상:** 용어사전 v1.3 의 핵심 신규 개념 18종이 6개 후속 문서(DE11-1 v1.2 / DE22-1 v1.5 메인+sections 8 / DE24-1 v1.8 / DE32-1 v1.0 / DE35-1 v1.5 / AN12-1 v1.1)에 누락 없이 반영됐는지 매트릭스 검증.
> - **전체 셀:** 18 개념 × 6 문서(열) = **108 셀**. (DE22-1 sections 는 8 파일을 묶어 1열로 집계, "범위에 해당하면 한 곳이라도 반영" 기준)
> - **결과 분포:** ✅ 84 (77.8%) / ⚠️ 7 (6.5%) / ❌ 5 (4.6%) / N/A 12 (11.1%)
> - **유효 셀(N/A 제외, 96)** 기준 반영률: **✅ 87.5% / ⚠️ 7.3% / ❌ 5.2%**
> - **결론:** **conditional pass** — Blocker 는 없고 P0 도 없음. P1·P2 5건만 보완하면 full pass.

## 2. 누락(❌) · 부분(⚠️) 항목 상세

### 2.1 ❌ 누락 5건

#### N1. DIES_BOOK·DIES_SUPPLIER·ITEM_SUPPLIER 화면 미작성 (DE22-1) — **개념 7, 14**
- **현황:** AN12-1 v1.1 line 576 (`SCR-PM-DIES-BOOK`, `SCR-PM-SUPPLIER` 식별자) 와 line 442~458 (FR-PM-023 다이스북 개정판·공급망 관리) 가 화면을 요구.
- **확인:** `docs/3_DE(설계)/DE22-1_화면설계서/sections/04_제품관리.md` 에서 `다이스북`·`DIES_BOOK`·`SCR-PM-DIES`·`SCR-PM-SUPPLIER` 검색 시 매칭 0건. sections 8개 어디에도 화면 정의 없음.
- **영향:** Phase 1 PM 서브시스템 화면 리스트 누락. 발주관리(OM) 가 공급사 마스터를 참조해야 하나 UI 없음.
- **보완:** DE22-1 sections/04_제품관리.md (혹은 별도 sections/08_다이스북공급망.md) 에 SCR-PM-018(다이스북 개정 이력), SCR-PM-019(공급사·역할), SCR-PM-020(자재-공급사 다대다) 3 화면 신설. (P1)

#### N2. modelCode 디코딩 세그먼트 정의 미언급 (DE32-1) — **개념 15**
- **현황:** DE32-1 v1.0 line 74 — `product_code VARCHAR(32) UNIQUE / modelCode. 예: DHS-AE225-D-1`. 단 한 줄 컬럼 정의만 있음. Brand/ProductType/Size/Hidden/Glass/Derivative 6 세그먼트 분해는 ER 다이어그램 범위 밖이라는 입장은 가능하지만, **modelCode 가 비결합 식별자가 아닌 의미적 코드** 임이 ER 주석에서 사라지면 후속 개발자에게 정보 손실.
- **영향:** 낮음 (DE35-1·AN12-1 에 디코딩 정의 있음). 그러나 `제품 검색·필터 인덱스 설계` 에서 세그먼트 prefix 인덱스 제안이 빠짐.
- **보완:** §2.2 PRODUCT 표 비고 셀에 "세그먼트 디코딩 가능, [[WIMS_용어사전_BOM_v1.3]] §15 참조" 1줄 추가. (P3)

#### N3. optionsHash 사용자 노출 정책 (DE22-1) — **개념 8**
- **현황:** DE22-1 v1.5 sections/05_BOM관리.md 에서 `optionsHash`·`appliedOptionsHash`·`NUMERIC 해시 제외` 검색 0건. 확정구성표 화면(line 413~448) 에 `resolvedBomId` 표시는 있지만 ENUM-only 해시 산출 규칙은 사용자에게 노출되지 않음.
- **영향:** 중간. UI 가 W=1500 과 W=1501 을 같은 resolvedBomId 로 보여주면 사용자 혼란. "왜 같은 ID 인가" 안내 툴팁/뱃지 부재.
- **보완:** 확정구성표 헤더에 "ID 산출 기준: ENUM 옵션만 해시. NUMERIC 치수는 별도 snapshot" 안내 추가. (P2)

## 3. 가장 심각한 누락 TOP 5

1. **N1. DE22-1 다이스북·공급망 화면 미작성** — AN12-1 FR-PM-023 가 요구하고 식별자(`SCR-PM-DIES-BOOK`, `SCR-PM-SUPPLIER`) 까지 부여했으나 화면 정의 자체가 sections 에 부재. **요구↔설계 추적 단절**. (P1)
2. **N3. DE22-1 optionsHash 노출 정책 부재** — 사용자가 `resolvedBomId` 동일성에 혼란 가능. 안내 카피 1줄로 해결되지만 누락 시 운영 문의 빈발 예상. (P2)
3. **⚠4. DE32-1 optionsHash 산출 규칙 본문 누락** — UNIQUE 인덱스만 있고 산출 규칙은 외부 문서 참조에 의존. ER 문서 자기완결성 약화. (P2)
4. **⚠1. DE24-1 OPTION_VALUE value_type/numeric_min DTO 미노출** — MES 가 옵션 마스터를 조회할 일은 적으나, 외부 검증 통합 시 필요. (P2)
5. **N2. DE32-1 modelCode 세그먼트 디코딩 주석 부재** — 정보 손실. 1줄 추가로 해결. (P3)

## 4. 결론

> [!success] 판정: **conditional pass**
> - **Blocker/P0 없음.** 18 개념 모두 최소 1개 후속 문서에 반영. 핵심 6 문서 평균 반영률 87.5%.
> - **DE11-1 v1.2 / DE35-1 v1.5 는 100%** — 구조 설계와 도메인 예제는 v1.3 을 완벽 흡수.
> - **DE32-1 v1.0** 은 ER 도구 문서로 산출 규칙·디코딩 등 본문 서술이 약함(77.8%) — 자기완결성 보강 필요.
> - **DE22-1 v1.5 sections** 가 다이스북·공급망 화면 누락(N1) 으로 PM 서브시스템 화면 셋 미완. **유일한 P1.**
> - **DE24-1 v1.8** 은 외부 API 범위상 9 셀이 N/A 인 점 감안 시 누락 0건, 부분 3건 모두 의도된 캡슐화 또는 P3 권장 수준.
> - **AN12-1 v1.1** 은 FR/NFR 모두 v1.3 18 개념을 추적표 형태(line 562~582) 로 완비.
> - **권장 액션 순서:** ① N1(다이스북 화면 신설, P1) → ② N3+⚠4(optionsHash 노출/명세, P2) → ③ ⚠1(DE24-1 OPTION_VALUE DTO, P2) → ④ 잔여 P3 일괄.

## 5. 5줄 요약

1. v1.3 핵심 18 개념 × 6 문서 = 108 셀 매트릭스 결과 ✅ 84 / ⚠️ 7 / ❌ 5 / N/A 12. 유효셀 반영률 **87.5%**.
2. **DE11-1·DE35-1 100% 완비**, AN12-1 88%, DE22-1·DE32-1 77.8%, DE24-1 67% (N/A 다수).
3. **유일한 P1 누락:** DE22-1 sections 에 다이스북·공급망 3 화면(SCR-PM-018~020) 미작성 — AN12-1 FR-PM-023 와 추적 단절.
4. **P2 보완 3건:** DE22-1 optionsHash 사용자 안내, DE32-1 optionsHash 산출 규칙 본문, DE24-1 OPTION_VALUE DTO.
5. **판정 conditional pass** — Blocker/P0 없음, P1 1건만 처리하면 full pass 도달.
